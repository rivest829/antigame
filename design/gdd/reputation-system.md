# 声誉系统 (Reputation System)

> **Status**: In Design
> **Author**: [user + agents]
> **Last Updated**: 2026-04-27
> **Implements Pillar**: 成长是可见的

## Overview

声誉系统是《地下排练室》的**进度量化与解锁驱动中枢**，负责记录乐队在地下音乐圈中的知名度，将玩家的每一次演出、每一个决策转化为可累积的"声誉值"。声誉不仅是数字——它是乐队从"无人知晓的地下室乐团"成长为"被记住的声音"的具象化证明，也是进度解锁系统判断"玩家是否准备好迎接下一阶段"的核心依据。

**系统边界**：
- **包含**：声誉值的获取与消耗、声誉等级划分、声誉收益计算、声誉衰减（可选）、声誉历史记录
- **不包含**：具体解锁内容的定义（属于内容数据库）、解锁条件的判定逻辑（属于进度解锁系统）、演出判定的具体机制（属于演出判定系统）、声誉的叙事包装（属于叙事事件系统）

**玩家如何感知**：
- 成功演出后声誉条上涨 → "我们在圈子里有点名字了"
- 解锁新场地需要达到特定声誉 → "要达到这个水平才能去那里演出"
- 声誉等级名称变化（"无名之辈" → "本地新声" → "圈内传说"） → "身份在转变"
- 失败演出导致声誉增长停滞 → "一次失败不会毁了名声，但也不会让它增长"

**为什么必须存在**：
没有声誉系统，玩家的成长没有统一的度量衡。技能涨了、曲目熟了、场地解锁了——但这些是分散的成就。声誉系统把它们编织成一条连续的弧线，让玩家清楚地看到"我们走到哪里了"。它是"成长是可见的"支柱的核心载体。

## Player Fantasy

> **"一开始，台下只有你的朋友。后来，是几个好奇的陌生人。再后来，你看见有人举着你们的名字。最后——你不需要看台下，你知道他们在那里。"**

你记得第一次演出。台下 5 个人，3 个是你们的朋友，1 个是场地老板，还有 1 个是误打误撞走进来的学生。你们演完，场地老板说："还行，下次再来。"你的声誉涨了 2 点。

你现在看着声誉条——已经 47 点了，等级是"本地新声"。你解锁了校园开放舞台，下一次演出将有 50 个观众。你不知道他们是谁，但他们知道你们的名字——至少听说过。

你想起了某个夜晚。排练结束后，林野说："我们要不要试试写一首自己的歌？"大伟说："等我们有 100 点声誉吧——那时候我们就有资格说自己是在做音乐了。"

那一刻你意识到：**声誉不是数字，是底气。**

**核心情感时刻**：
当你从"无名之辈（0-20）"升到"本地新声（21-50）"时，UI 不是弹出一个恭喜通知。而是触发了一个叙事事件——有个本地音乐博主写了篇关于你们的文章。你读那篇文章，里面说："这支乐队的声音像生锈的铁链在水泥地上拖动——粗糙，但有力。"你看着林野的 reaction（他假装不在乎，但嘴角在动），你知道：**这个世界开始注意我们了。**

**参考体验**：
- 《星露谷物语》的社区中心进度——每个房间完成都是小镇对玩家的认可
- 《偶像梦幻祭》的粉丝数——从小圈子到大舞台，数字背后是叙事

**这不是"刷分"的幻想——这是"被看见"的幻想。**

## Detailed Design

### Core Rules

#### 1. 声誉值 (Reputation Points)

声誉是一个整数，范围 0 到 `max_reputation`（MVP 设定为 200）。

**获取途径**：

| 来源 | 条件 | 基础声誉值 | 说明 |
|------|------|-----------|------|
| **演出成功** | 演出判定为"成功"或以上 | 见演出收益公式 | 主要来源 |
| **演出完美** | 演出判定为"完美" | 2× 基础值 | 超额奖励 |
| **首次解锁场地** | 解锁新场地后首次演出 | +5 额外声誉 | 探索奖励 |
| **配合技触发** | 演出中触发配合技 | +3 额外声誉 | 化学反应奖励 |
| **叙事事件** | 某些事件的结果 | 事件定义 | 故事驱动 |

**不会减少声誉的情况**：
- 演出失败：声誉不减少，但本轮不获得声誉
- 角色争吵：可能触发负面事件，但事件本身可能减少声誉（通过事件系统）

**设计意图**：声誉是单向累积的（或至少不会直接因演出失败而倒退）。失败的机会成本是"浪费了时间却没有成长"，而不是"被惩罚"。这符合"节奏是轻松的"支柱。

#### 2. 声誉等级 (Reputation Tiers)

声誉划分为 5 个等级，每个等级有名称、图标和解锁含义：

| 等级 | 声誉范围 | 名称 | 图标 | 解锁含义 |
|------|---------|------|------|---------|
| 1 | 0-20 | 无名之辈 | 空白头像 | 仅地下室可用 |
| 2 | 21-50 | 本地新声 | 1 颗星 | 解锁校园开放舞台 |
| 3 | 51-100 | 圈内传说 | 2 颗星 | 解锁本地 Livehouse |
| 4 | 101-150 | 城市名片 | 3 颗星 | 解锁音乐节 |
| 5 | 151-200 | 传奇乐队 | 4 颗星 | 解锁万人场馆 |

**等级提升的叙事包装**：
每次升级不弹出通用通知，而是触发一个独特的叙事事件：
- 0→21：有人在网上提到了你们的乐队
- 21→51：本地音乐杂志做了小篇幅报道
- 51→100：有经纪人开始打听你们
- 101→150：音乐节主办方发来了邀请
- 151→200：万人场馆的档期表上出现了你们的名字

#### 3. 演出声誉收益计算

演出获得的声誉由以下公式决定：

```
reputation_gained = base_reward × performance_multiplier × difficulty_multiplier × venue_multiplier
```

**Base Reward (基础奖励)**：
- 来自内容数据库的 `reputation_reward_base`
- 地下室：5
- 校园：8
- Livehouse：12
- 音乐节：18
- 万人馆：25

**Performance Multiplier (演出表现倍率)**：

| 演出评级 | 倍率 | 说明 |
|---------|------|------|
| 灾难 | 0 | 失败，无声誉 |
| 平庸 | 0.5 | 勉强完成 |
| 成功 | 1.0 | 标准完成 |
| 出色 | 1.5 | 超预期 |
| 完美 | 2.0 | 无可挑剔 |

**Difficulty Multiplier (难度倍率)**：

```
difficulty_multiplier = 1.0 + (final_difficulty / 100) × 0.5
```

- 难度 0（轻松）→ 1.0×
- 难度 50（适中）→ 1.25×
- 难度 100（疯狂）→ 1.5×

**设计意图**：挑战更高难度的演出获得更高声誉回报。鼓励玩家冒险。

**Venue Multiplier (场地倍率)**：
- 来自内容数据库的 `reward_multiplier`
- 与 `difficulty_modifier` 通常正相关——高 tier 场地高收益

#### 4. 声誉上限与软上限

**硬上限**：`max_reputation = 200`（MVP）

**软上限机制**：
当声誉接近上限时，低难度演出的收益递减：

```
soft_cap_penalty = max(0, (current_reputation - 150) / 50)
# 0-150: penalty = 0
# 151-200: penalty = 0.02 to 1.0
```

当声誉 > 150 时：
```
reputation_gained = reputation_gained × (1.0 - soft_cap_penalty)
```

**设计意图**：防止玩家通过反复刷低难度场地达到顶级声誉。要到达"传奇乐队"，必须挑战高难度演出。

#### 5. 声誉历史

系统记录最近 20 次声誉变化：

```json
{
  "reputation_history": [
    {
      "timestamp": 1713800000,
      "change": +12,
      "reason": "performance_success",
      "venue": "venue_campus_stage",
      "song": "rust",
      "rating": "出色"
    }
  ]
}
```

**用途**：
- 玩家查看"我们是怎么走到这里的"
- 调试和平衡分析
- 叙事事件引用（"记得那场演出吗？"）

#### 6. 声誉与观众期待值（联动演出难度系统）

```
audience_expectation = 1.0 + (reputation / max_reputation) × 0.3
```

声誉越高，观众期待越高，相同客观难度下"感知难度"增加。这是声誉系统的"甜蜜的负担"——你出名了，观众不再宽容。

### States and Transitions

声誉系统本身是无状态的累加器，但在游戏进程中有以下阶段：

| 阶段 | 触发条件 | 系统行为 |
|------|---------|---------|
| **积累** | 演出成功、事件奖励 | 增加声誉值 |
| **升级** | 声誉跨越等级阈值 | 触发等级提升叙事事件 |
| **解锁检查** | 声誉变化后 | 通知进度解锁系统检查新解锁 |
| **封顶** | 声誉达到 200 | 显示"传奇乐队"状态，低难度收益递减 |

## Formulas

### 1. 演出声誉收益公式 (Performance Reputation Gain)

```
reputation_gained = floor(base_reward × performance_multiplier × difficulty_multiplier × venue_multiplier × (1.0 - soft_cap_penalty))
```

**Variables:**
| Variable | Symbol | Type | Range | Description |
|----------|--------|------|-------|-------------|
| base_reward | br | int | 5-25 | 场地基础声誉奖励（来自内容数据库） |
| performance_multiplier | pm | float | 0.0-2.0 | 演出评级倍率 |
| difficulty_multiplier | dm | float | 1.0-1.5 | 演出难度倍率 |
| venue_multiplier | vm | float | 0.8-2.0 | 场地收益倍率（来自内容数据库） |
| soft_cap_penalty | scp | float | 0.0-1.0 | 软上限惩罚 |

**Performance Multiplier Values:**
| Rating | Value |
|--------|-------|
| 灾难 (Disaster) | 0.0 |
| 平庸 (Mediocre) | 0.5 |
| 成功 (Success) | 1.0 |
| 出色 (Great) | 1.5 |
| 完美 (Perfect) | 2.0 |

**Difficulty Multiplier Sub-formula:**
```
difficulty_multiplier = 1.0 + (final_difficulty / 100) × 0.5
```

**Soft Cap Penalty Sub-formula:**
```
soft_cap_penalty = max(0, (current_reputation - 150) / 50)
```

**Output Range:** 0 to 75 (before floor)
**Example:** 校园舞台 (base=8), 出色演出 (pm=1.5), 难度 40 (dm=1.2), venue=1.0, reputation=60 (scp=0)
→ `gain = floor(8 × 1.5 × 1.2 × 1.0 × 1.0) = floor(14.4) = 14`

**Example 2 (软上限):** 同上，但 reputation=180 (scp=0.6)
→ `gain = floor(14.4 × 0.4) = floor(5.76) = 5`

---

### 2. 声誉等级判定公式 (Reputation Tier)

```
tier = clamp(ceil(reputation / tier_threshold), 1, 5)
```

**Tier Thresholds:**
| Tier | Range | Threshold |
|------|-------|-----------|
| 1 | 0-20 | 20 |
| 2 | 21-50 | 50 |
| 3 | 51-100 | 100 |
| 4 | 101-150 | 150 |
| 5 | 151-200 | 200 |

**Output Range:** 1 to 5
**Example:** reputation=47 → `tier = ceil(47/50) = 1` (but within 21-50 range = tier 2)

*Note: Tier is determined by range check, not formula. The formula is for programmatic lookup.*

---

### 3. 观众期待值公式 (Audience Expectation)

```
audience_expectation = 1.0 + (reputation / max_reputation) × 0.3
```

**Variables:**
| Variable | Symbol | Type | Range | Description |
|----------|--------|------|-------|-------------|
| reputation | rep | int | 0-200 | 当前声誉值 |
| max_reputation | max | int | 200 | 声誉上限 |

**Output Range:** 1.0 to 1.3
**Example:** reputation=100 → `expectation = 1.0 + (100/200) × 0.3 = 1.15`

---

### 4. 声誉进度百分比公式 (Progress to Next Tier)

```
tier_progress = (reputation - tier_min) / (tier_max - tier_min) × 100
```

**Variables:**
| Variable | Symbol | Type | Range | Description |
|----------|--------|------|-------|-------------|
| reputation | rep | int | 0-200 | 当前声誉值 |
| tier_min | min | int | 0-150 | 当前等级下限 |
| tier_max | max | int | 20-200 | 当前等级上限 |

**Output Range:** 0% to 100%
**Example:** reputation=35 (tier 2: 21-50) → `progress = (35-21) / (50-21) = 14/29 = 48.3%`

## Edge Cases

- **If 演出评级为"灾难"**: `performance_multiplier = 0`，本轮无声誉收益。但声誉不减少。玩家失去的是时间成本和机会成本。
- **If 声誉已达到 max_reputation (200)**: 继续演出仍可提升其他指标（如熟练度、角色关系），但声誉值不再增加。软上限惩罚=1.0，所有声誉收益为0。
- **If 声誉因事件系统而减少**: 允许减少到0以下吗？不，声誉下限为0。如果事件试图减少超过当前值，减少量被截断到当前值。
- **If 声誉跨越多个等级阈值**: 一次大额增加可能跨越多个等级。系统依次触发每个等级的叙事事件（例如从18→55，依次触发"本地新声"和"圈内传说"事件）。
- **If 演出判定系统尚未实现**: 声誉系统使用模拟的演出评级进行测试。实际集成后接入真实评级。
- **If 内容数据库中的场地参数缺失**: 使用默认值 `base_reward=5`, `reward_multiplier=1.0`。记录警告日志。
- **If 玩家反复刷同一低难度场地**: 软上限机制（reputation > 150后收益递减）+ 场地本身低基础奖励 = 刷级效率极低。设计意图：鼓励挑战更高难度。
- **If 声誉历史超过20条**:  oldest entry removed, new entry appended. FIFO队列。
- **If 演出时声誉刚好跨越阈值**: 以演出前的声誉值计算观众期待值，演出后的声誉值用于判定是否升级。避免同一演出既享受低期待又获得升级奖励。
- **If 配合技触发增加声誉**: 多次配合技在同一演出中触发 = 多次+3奖励，上限+9（避免极端情况）。

## Dependencies

### 输入依赖（本系统需要的数据）

| 来源系统 | 输入数据 | 作用 |
|----------|----------|------|
| 内容数据库 | 场地 `reputation_reward_base`、`reward_multiplier` | 计算演出声誉收益 |
| 演出判定系统 | 演出评级（灾难/平庸/成功/出色/完美） | 确定 `performance_multiplier` |
| 演出难度系统 | `final_difficulty`（0-100） | 计算 `difficulty_multiplier` |
| 化学反应系统 | 配合技触发信号 | 额外声誉奖励 |
| 叙事事件系统 | 事件结果（可能奖励/扣除声誉） | 非演出来源的声誉变化 |
| 存档/加载系统 | 声誉值持久化 | 读写 `reputation` 字段 |

### 输出依赖（使用本系统数据的系统）

| 目标系统 | 输出数据 | 作用 |
|----------|----------|------|
| 进度解锁系统 | 当前声誉值、声誉等级 | 判定内容解锁条件 |
| 演出难度系统 | `audience_expectation`（1.0-1.3） | 高声誉增加感知难度 |
| 叙事事件系统 | 等级提升事件触发 | 声誉升级时触发叙事事件 |
| 主界面UI | 声誉值、等级、进度百分比 | 显示乐队知名度 |

### 双向依赖说明

- **演出难度系统**：声誉 → 观众期待值 → 难度增加。这是设计意图的"甜蜜的负担"——成功带来更高期待，更高期待带来更大挑战。
- **进度解锁系统**：声誉等级是解锁的核心条件之一。双向弱依赖通过快照避免：解锁检查使用当前声誉值，不依赖历史。

### 依赖顺序

```
内容数据库 ──┐
演出判定系统 ──┼──→ 声誉系统 ──→ 进度解锁系统
演出难度系统 ──┤      ↓
化学反应系统 ──┘   叙事事件系统
```

## Tuning Knobs

| 参数 | 当前值 | 安全范围 | 调整影响 | 所在公式 |
|------|--------|---------|---------|---------|
| `max_reputation` | 200 | 150-500 | 达到顶级的总工作量。越高=游戏越长 | 全局上限 |
| `tier_2_threshold` | 21 | 15-30 | 解锁第一个新场地的速度 | 等级阈值 |
| `tier_3_threshold` | 51 | 40-60 | 解锁 Livehouse 的速度 | 等级阈值 |
| `tier_4_threshold` | 101 | 80-120 | 解锁音乐节的速度 | 等级阈值 |
| `tier_5_threshold` | 151 | 130-170 | 解锁万人馆的速度 | 等级阈值 |
| `performance_perfect_multiplier` | 2.0 | 1.5-3.0 | 完美演出的奖励强度。越高=追求完美的动力越强 | `performance_multiplier` |
| `difficulty_multiplier_max` | 1.5 | 1.2-2.0 | 最高难度演出的额外收益。越高=冒险越值得 | `difficulty_multiplier` |
| `soft_cap_start` | 150 | 120-180 | 软上限开始生效的声誉值。越低=后期越早需要挑战高难度 | `soft_cap_penalty` |
| `audience_expectation_max` | 0.3 | 0.1-0.5 | 顶级声誉对难度的加成幅度。越高=高声誉越难维持 | `audience_expectation` |
| `combo_bonus_reputation` | 3 | 1-5 | 每次触发配合技的额外声誉。越高=化学反应越重要 | 额外奖励 |
| `combo_bonus_max_per_performance` | 9 | 5-15 | 单场演出配合技声誉上限 | 额外奖励 |

**调优策略**：
- 如果玩家升级太快：提高阈值或降低 `performance_perfect_multiplier`
- 如果玩家不敢挑战高难度：提高 `difficulty_multiplier_max`
- 如果后期玩家感觉"刷不动了"：降低 `soft_cap_start` 或提高 `max_reputation`
- 如果玩家只刷一个场地：提高低 tier 场地的软上限惩罚或降低其 `base_reward`

## Visual/Audio Requirements

### 视觉需求

**声誉条**：
- 位置：主界面仪表盘顶部，角色状态卡下方
- 样式：水平进度条，背景为深灰色，填充色随等级变化：
  - 等级1（无名之辈）：暗褐色 #5C4033
  - 等级2（本地新声）：暗金色 #B8860B
  - 等级3（圈内传说）：铜色 #CD7F32
  - 等级4（城市名片）：银色 #C0C0C0
  - 等级5（传奇乐队）：金色 #FFD700
- 动画：声誉增加时，填充部分有"粒子溢出"效果（像火花从条中溅出）
- 文字：当前值 / 下一级阈值（如 "47 / 51"）

**等级提升动画**：
- 不是弹出窗口，而是主界面上的"报纸/海报"飘落动画
- 新等级名称以涂鸦风格出现在屏幕中央，停留 2 秒后淡出
- 伴随解锁内容图标（如新场地）从屏幕边缘滑入

**声誉历史界面**：
- 一条垂直时间线，每个节点是一个演出事件
- 节点大小与声誉变化量成正比
- 正变化：向上生长的线条（绿色/金色）
- 负变化：向下凹陷的线条（红色）

### 音频需求

**声誉增加音效**：
- 小幅增加（1-5）：轻快的木琴音符
- 中幅增加（6-15）：小鼓滚奏
- 大幅增加（16+）：钹的强击 + 观众欢呼（短）

**等级提升音效**：
- 电吉他强力和弦（失真效果）
- 持续时间 1-2 秒，与报纸飘落动画同步

**声誉达到上限**：
- 特殊的"终章"音效——不是庆祝，而是一种"沉淀"的感觉
- 暗示"我们已经到了，现在要保持"

## UI Requirements

### 主界面仪表盘

**声誉卡片**：
- 位置：屏幕顶部中央，仅次于标题的显著位置
- 内容：
  - 当前等级图标 + 等级名称
  - 声誉进度条（当前值 / 下一级阈值）
  - 简短描述（如 "还差 4 点声誉升级"）
- 交互：点击展开声誉历史时间线

**声誉历史界面**：
- 模态面板，从底部滑出
- 垂直时间线，最近 20 次声誉变化
- 每条记录显示：时间、变化量、原因、场地、曲目
- 筛选选项：全部 / 仅演出 / 仅事件

### 演出结果界面

**声誉收益显示**：
- 演出结果弹窗中，声誉变化单独一行
- 显示：基础奖励 × 表现倍率 × 难度倍率 = 最终收益
- 如果触发了软上限惩罚：显示惩罚图标和说明"接近传奇地位，普通演出收益递减"

**升级提示**：
- 如果本次演出导致升级：在结果界面顶部显示大号的等级提升动画
- 显示新解锁内容（如"解锁场地：本地 Livehouse"）
- 提供"查看新内容"按钮，直接导航到解锁内容界面

### 演出安排界面

**声誉收益预览**：
- 在难度预览旁边，显示"预计声誉收益"
- 基于当前难度和场地参数计算
- 显示范围（如 "8-16"，基于可能的演出评级波动）

## Acceptance Criteria

### 功能测试

- [ ] 演出成功后声誉正确增加（计算符合公式）
- [ ] 演出失败后声誉不增加也不减少
- [ ] 完美演出获得 2× 基础声誉
- [ ] 难度越高的演出，声誉收益越高（难度倍率生效）
- [ ] 声誉跨越阈值时正确升级并触发叙事事件
- [ ] 声誉达到 200 后不再增加
- [ ] 声誉 > 150 后低难度演出收益递减（软上限生效）
- [ ] 配合技触发增加 +3 声誉，单场上限 +9

### 边界测试

- [ ] 声誉为 0 时演出成功 → 正确增加
- [ ] 声誉接近 200 时演出成功 → 软上限正确截断
- [ ] 事件试图将声誉减到负数 → 截断到 0
- [ ] 一次跨越多个等级阈值 → 依次触发所有等级事件
- [ ] 内容数据库参数缺失 → 使用默认值，记录日志

### 集成测试

- [ ] 从演出判定系统接收评级 → 正确计算声誉
- [ ] 从演出难度系统接收 difficulty → 正确计算难度倍率
- [ ] 输出观众期待值到演出难度系统 → 值在 1.0-1.3 范围
- [ ] 声誉升级 → 正确通知进度解锁系统
- [ ] 声誉值正确保存/加载（存档系统）

### 性能测试

- [ ] 声誉计算耗时 < 1ms
- [ ] 声誉历史查询（20条）耗时 < 5ms

## Open Questions

- **声誉衰减是否实现？** 当前设计是单向累积（或至少不会因演出失败而倒退）。是否需要"长期不演出导致声誉缓慢下降"的机制？这可能增加紧迫感，但也可能让玩家焦虑。
- **声誉的叙事包装颗粒度？** 每个等级提升都有独特事件（5个）。是否需要中间节点的事件（如声誉达到 30、75 等）？
- **不同音乐风格是否影响声誉？** 例如演出一首朋克曲目在朋克圈子获得额外声誉？MVP 不需要，但未来扩展呢？
- **声誉与角色个人声望的关系？** 乐队声誉和角色个人声誉是否分离？例如林野个人在吉他圈有名声，但乐队整体声誉不同？
- **线上/线下声誉区分？** 现场演出声誉 vs 网络传播声誉（社交媒体、流媒体）？MVP 统一为"声誉"，但未来可能拆分。
- **声誉是否影响排练收益？** 高声誉乐队排练时是否获得额外动力（如"我们有名了，不能松懈"）？或者 NPC 角色更愿意加入高声誉乐队？
