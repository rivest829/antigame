# 主界面/仪表盘UI (Main Dashboard UI)

> **Status**: Designed (pending review)
> **Author**: [user + agents]
> **Last Updated**: 2026-04-29
> **Implements Pillar**: 成长是可见的, 节奏是轻松的

## Overview

主界面/仪表盘UI是《地下排练室》的**游戏大厅**——玩家每次打开游戏后看到的第一个屏幕，也是所有功能的总入口。它需要在5秒内让玩家理解"乐队当前状态"，在15秒内让玩家完成"收成果→做决策→设计划"的核心循环。它是"成长是可见的"和"节奏是轻松的"两大支柱的首要呈现层。

**系统边界**：
- **包含**：主界面布局、离线收益展示动画、角色状态概览、排练状态显示、事件通知红点、导航入口、快捷操作
- **不包含**：各子系统的详细UI（属于各自UI系统）、排练计划的具体设置（属于排练计划UI）、演出结果的具体展示（属于演出结果UI）

**玩家如何感知**：
- 打开游戏，地下室灯光慢慢亮起 → "我回来了"
- 数字从0滚动到实际值 → "他们在成长"
- 红点提示"2条新消息" → "发生了一些事"
- 所有操作都能在3次点击内完成 → "轻松"

**为什么必须存在**：
没有主界面UI，玩家登录后面对的是功能的海洋——不知道该先看什么、该做什么。主界面UI是信息的"策展人"，它决定哪些信息最重要、什么时候展示、用什么方式展示。它是玩家体验的门面，也是留存的关键：如果每次登录都有值得期待的新变化，玩家就会养成"打开看看"的习惯。

## Player Fantasy

> **你打开手机。屏幕先是一片漆黑。**
>
> 然后，像有人按下了地下室的开关，一盏昏黄的灯泡在屏幕中央慢慢亮起。你开始看到轮廓：墙角堆着的音箱、地板上散乱的线缆、窗台上那个永远种不活的仙人掌。
>
> 灯光完全亮起后，你看到林野趴在音箱上睡觉，头发乱糟糟的。大伟坐在鼓后面，正在调踩镲的高度。没有人说话——地下排练室总是这样安静的。
>
> 然后，屏幕上方出现了一行字："离线4小时。"像有人用马克笔在墙上写的。
>
> 数字开始滚动：
> - 《锈》熟练度：45% → 57%（+12%）
> - 林野·吉他：42 → 48（+6）
> - 大伟·鼓：31 → 35（+4）
> - 林野精力：60 → 28（-32）
> - 大伟精力：55 → 35（-20）
>
> 数字滚动时有轻微的"咔嗒"声，像有人在用老式计数器。
>
> 滚动结束后，右上角的红点开始跳动："2条新消息"。
>
> 你点击了红点。屏幕上浮现出两张皱巴巴的纸条——林野和大伟的争执、深夜的排练室。
>
> 处理完事件后，你回到主界面。林野和大伟的头像旁边多了一些小字："林野心情下降"、"大伟状态正常"。
>
> 你看了一眼排练状态区：《锈》的排练已经完成，显示"点击收取"。你点了下去，屏幕上闪过一道光——排练成果已确认。
>
> 然后你点了"设置排练"，进入排练计划UI。
>
> **整个流程不超过2分钟。你没有感到任何压力。**
>
> 核心情感时刻：当你连续登录7天，主界面的背景会 subtly 变化——墙上的海报从1张变成3张，窗台的仙人掌长出了一片新叶子，地板上的线缆变得稍微整齐了一点。这些变化不会有人告诉你，但你会发现。你会感到：**这支乐队在和我一起成长。**

**这不是"管理"的幻想——这是"回家"的幻想。** 你打开游戏，不是为了完成什么任务，而是为了看看"他们最近怎么样"。

## Detailed Design

### Core Rules

#### 1. 界面布局（五区纵向布局）

主界面采用垂直滚动布局，从上到下依次为：

**A区：顶部状态栏**
- 位置：屏幕顶部（固定，不随滚动消失）
- 内容：
  - 乐队名称（可编辑，默认"未命名乐队"）
  - 当前日期/时间（游戏内时间）
  - 设置按钮（齿轮图标）
  - 通知图标（铃铛，有红点时跳动）

**B区：声誉与进度**
- 位置：顶部下方
- 内容：
  - 声誉进度条（当前值/下一级阈值）
  - 声誉等级名称和图标
  - 点击展开声誉历史时间线

**C区：角色状态卡片区**
- 位置：屏幕中上部
- 内容：每个角色的状态卡片
- 卡片内容：头像、姓名、主乐器、技能值、精力条、心情图标
- 点击卡片进入角色详情UI

**D区：排练状态区**
- 位置：屏幕中部
- 状态显示：
  - **无排练计划**："没有正在进行的排练" + "设置排练"按钮
  - **排练中**：曲目名称、进度条、预计完成时间
  - **排练完成**："排练完成，点击收取" + 闪烁提示
- 点击区域进入排练计划UI

**E区：快捷操作与导航区**
- 位置：屏幕下部
- 内容：
  - 快捷按钮：设置排练 / 安排演出 / 查看历史
  - 底部导航栏：主界面 / 乐队 / 演出 / 设置（4个Tab）

#### 2. 离线收益展示

**触发时机**：玩家登录后，如果检测到离线期间有收益

**展示流程**：
```
1. 屏幕渐暗 → 显示"欢迎回来"文字
2. 显示离线时长
3. 收益数字逐个出现，从0滚动到实际值
4. 每个数字滚动持续1秒，间隔0.3秒
5. 所有数字滚动完成后，显示"点击继续"
6. 玩家点击后，恢复正常主界面
```

**展示内容**（按顺序）：
1. 曲目熟练度变化（如果有）
2. 角色技能成长（如果有）
3. 角色状态变化（精力/心情）
4. 声誉变化（如果有演出）
5. 新解锁内容（如果有）
6. 新事件数量（如果有）

**跳过机制**：点击屏幕任意位置跳过当前动画，直接进入主界面

#### 3. 角色状态卡片

**卡片设计**：
```
┌─────────────────────────────┐
│ ┌────┐ 林野                  │
│ │头像│ 吉他手                │
│ └────┘ ┌──────────┐         │
│        │技能: 48  │         │
│        └──────────┘         │
│ 精力 [████████░░░░] 68%     │
│ 心情 [😊] 亢奋               │
└─────────────────────────────┘
```

**卡片状态**：
- 正常状态：正常显示
- 精力≤20：卡片边缘红色闪烁，头像变暗
- 心情低落：头像表情变为😔，卡片色调偏蓝
- 心情亢奋：头像表情变为😄，卡片有轻微发光效果

**卡片交互**：
- 点击：进入角色详情UI
- 长按：显示角色快捷操作（查看详情 / 查看技能树）

#### 4. 排练状态显示

**三种状态**：

| 状态 | 显示内容 | 交互 |
|------|---------|------|
| **无计划** | "乐队正在休息" + 角色待机动画 | 点击"设置排练"按钮 |
| **排练中** | 曲目封面小图 + 曲目名称 + 进度条 + "预计XX分钟后完成" | 点击可查看排练详情（不可中断） |
| **完成** | "排练完成！" + 闪烁光效 + "点击收取" | 点击收取成果，弹出排练成果弹窗 |

**排练中进度条**：
- 样式：像一根被慢慢烧焦的琴弦
- 颜色：从银白（0%）到铁锈红（100%）
- 实时更新：每5秒更新一次（与实时排练阶段性结算同步）

#### 5. 事件通知红点

**红点规则**：
- 有未读叙事事件时，通知图标显示红点
- 红点数字 = 未读事件数（最多显示"9+"）
- 红点动画：轻微脉动（缩放1.0→1.2→1.0，周期2秒）

**点击行为**：
- 点击红点/通知图标 → 进入叙事事件UI
- 处理完所有事件后，红点消失

#### 6. 导航入口

**底部导航栏**（4个Tab）：

| Tab | 图标 | 内容 | 未读标记 |
|-----|------|------|---------|
| 主界面 | 房子 | 当前页面 | 无 |
| 乐队 | 人物 | 角色详情/管理 | 角色状态异常时红点 |
| 演出 | 舞台 | 演出安排/历史 | 新演出可用时红点 |
| 设置 | 齿轮 | 设置菜单 | 无 |

**快捷操作按钮**（排练状态区下方）：
- 设置排练 → 排练计划UI
- 安排演出 → 演出安排UI
- 查看历史 → 历史记录UI

#### 7. 背景动态变化

**长期背景变化**（7天连续登录）：
- 第1天：墙上1张海报，地板杂乱
- 第3天：墙上2张海报，地板稍微整齐
- 第7天：墙上3张海报，地板整洁，窗台仙人掌长出新叶

**变化规则**：
- 变化是 subtle 的，不会突兀
- 变化与游戏进度挂钩（如解锁新场地后，墙上出现新场地的演出传单）
- 变化不可逆（即使断签，已变化的背景不会回退）

### States and Transitions

主界面UI有以下显示状态：

| 状态 | 说明 | 转换条件 |
|------|------|---------|
| **离线收益展示 (Offline_Reward)** | 登录后展示离线收益动画 | 检测到离线收益时进入 |
| **主界面正常 (Normal)** | 标准主界面 | 默认状态 |
| **排练收取 (Rehearsal_Collect)** | 排练完成，等待玩家收取 | 排练完成时进入 |
| **新解锁展示 (New_Unlock)** | 展示新解锁内容 | 解锁触发时进入 |
| **事件处理 (Event_Handling)** | 玩家正在处理叙事事件 | 点击红点时进入（跳转到叙事事件UI） |

```
登录 → 离线收益展示（如有） → 新解锁展示（如有） → 主界面正常
                                    ↓
                              排练收取（如有） → 主界面正常
                                    ↓
                              事件处理（玩家点击） → 主界面正常
```

## Formulas

### 1. 离线收益数字滚动动画 (Offline Reward Number Scroll)

```
display_value = lerp(0, actual_value, ease_out_cubic(t))
where t = elapsed_time / animation_duration
```

**Variables:**
| Variable | Symbol | Type | Range | Description |
|----------|--------|------|-------|-------------|
| animation_duration | dur | float | 800-1500ms | 单个数字滚动时长 |
| stagger_delay | delay | float | 200-400ms | 数字之间的间隔 |
| ease_function | ease | function | ease_out_cubic | 缓动函数 |

**Example**: 熟练度从45%到57%（+12%）
- t=0: display=0
- t=0.5: display≈10.5（缓动）
- t=1.0: display=12

---

### 2. 红点脉动动画 (Red Dot Pulse)

```
scale = 1.0 + sin(time × frequency) × amplitude
```

**Variables:**
| Variable | Symbol | Type | Range | Description |
|----------|--------|------|-------|-------------|
| frequency | freq | float | π rad/s | 脉动频率 |
| amplitude | amp | float | 0.1-0.2 | 脉动幅度 |

**Output**: scale在0.9-1.1之间平滑波动

## Edge Cases

- **If 离线时长<1分钟**：跳过离线收益展示，直接进入主界面。
- **If 离线期间无任何收益**（无排练计划）：显示"欢迎回来"+离线时长，无数字滚动，2秒后自动进入主界面。
- **If 多个系统同时需要展示**（离线收益+新解锁+排练完成）：按顺序展示，顺序为：离线收益 → 新解锁 → 排练完成。每个展示可独立跳过。
- **If 玩家首次登录（无存档）**：显示"欢迎来到地下排练室"引导界面，提供"创建乐队"入口，不显示离线收益。
- **If 角色状态极端异常**（如所有角色精力=0）：主界面显示紧急提示"乐队需要休息"，角色卡片红色闪烁。
- **If 排练中玩家将游戏切到后台再回来**：排练状态区显示最新进度（与实时排练同步）。
- **If 新解锁内容超过5个**：分批展示，每批最多3个，间隔1.5秒。
- **If 底部导航栏在横屏模式下**：导航栏移至屏幕左侧（垂直排列），内容区在右侧。

## Visual/Audio Requirements

### 视觉需求

**整体风格**：地下排练室的实时画面——不是静态图片，而是有微弱动态效果的场景（灯泡轻微闪烁、灰尘在光柱中飘动）。

**离线收益展示**：
- 背景：半透明黑色遮罩
- 文字：手写体马克笔风格
- 数字：从0滚动到实际值，字体粗大，颜色根据数值类型变化（绿色=成长，红色=消耗）
- 动画：数字出现时伴随轻微的"墨水晕染"效果

**角色卡片**：
- 头像：2D卡通风格，表情随心情变化
- 精力条：像一根被慢慢烧短的蜡烛，从满到空
- 心情图标：😊😔😠等emoji风格，但手绘化

**排练状态区**：
- 无计划：显示"乐队正在休息"，角色待机动画（林野趴桌、大伟调鼓）
- 排练中：进度条像烧焦的琴弦，从银白到铁锈红
- 完成：闪烁光效，像有人打开了闪光灯

**底部导航**：
- 选中Tab：图标放大，下方有墨迹线条
- 未选中Tab：图标正常，灰色
- 切换动画：墨迹线条滑动到选中项

### 音频需求

**登录音效**：
- 地下室灯泡"嗞"一声亮起（电流声）

**离线收益数字滚动**：
- 每个数字滚动：轻微的"咔嗒"声（老式计数器）
- 成长数值：音调较高
- 消耗数值：音调较低

**红点脉动**：
- 无声（视觉提示足够）

**排练完成**：
- 一声终于对准调的吉他扫弦

**Tab切换**：
- 轻微的"唰"声（像翻纸）

## Acceptance Criteria

### 功能测试

- **GIVEN** 离线4小时且有排练计划，**WHEN** 登录，**THEN** 显示离线收益动画，数字正确滚动
- **GIVEN** 有2个未读事件，**WHEN** 进入主界面，**THEN** 通知图标显示红点数字"2"
- **GIVEN** 排练完成，**WHEN** 进入主界面，**THEN** 排练状态区显示"排练完成，点击收取"
- **GIVEN** 点击排练状态区，**WHEN** 排练已完成，**THEN** 弹出排练成果弹窗
- **GIVEN** 点击角色卡片，**WHEN** 林野卡片被点击，**THEN** 进入林野的角色详情UI
- **GIVEN** 点击"设置排练"按钮，**WHEN** 按钮被点击，**THEN** 进入排练计划UI

### 边界测试

- **GIVEN** 离线<1分钟，**WHEN** 登录，**THEN** 跳过离线收益动画
- **GIVEN** 无排练计划且无离线收益，**WHEN** 登录，**THEN** 直接显示主界面
- **GIVEN** 所有角色精力=0，**WHEN** 进入主界面，**THEN** 显示紧急提示，角色卡片红色闪烁
- **GIVEN** 横屏模式，**WHEN** 进入主界面，**THEN** 导航栏在左侧，内容在右侧

### 集成测试

- **GIVEN** 从放置排练系统读取排练状态，**WHEN** 展示主界面，**THEN** 状态正确显示
- **GIVEN** 从叙事事件系统读取未读数，**WHEN** 展示主界面，**THEN** 红点数字正确
- **GIVEN** 从声誉系统读取声誉值，**WHEN** 展示主界面，**THEN** 进度条正确
- **GIVEN** 排练收取完成，**WHEN** 提交给放置排练系统，**THEN** 状态更新为"已收获"

### 性能测试

- **GIVEN** 离线收益展示，**WHEN** 数字滚动，**THEN** 帧率保持60fps
- **GIVEN** 主界面排练中状态，**WHEN** 实时更新进度条，**THEN** 每5秒更新，无卡顿
- **GIVEN** 点击导航Tab，**WHEN** 切换页面，**THEN** 切换动画<300ms

## Open Questions

- **Q1**: 主界面背景是否需要支持"自定义"——玩家可以更换墙上的海报、调整家具位置？这会增加沉浸感，但增加开发量。
- **Q2**: 是否需要"每日登录奖励"——连续登录的额外奖励？放置游戏常见，但可能破坏"节奏是轻松的"支柱。
- **Q3**: 角色待机动画是否需要更多变化——如林野有时在写歌、有时在调音、有时在睡觉？变化越多，世界越活。
- **Q4**: 是否需要"快捷操作"——在主界面直接拖拽角色到排练区，而不需要进入排练计划UI？
- **Q5**: 离线收益展示是否应该支持"一键跳过全部"——当前设计是每个展示可独立跳过，但玩家可能想一次性跳过所有。
