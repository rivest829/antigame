# 关系矩阵系统 (Relationship Matrix System)

> **Status**: In Design
> **Author**: [user + agents]
> **Last Updated**: 2026-04-24
> **Implements Pillar**: 人物是活的, 选择是有意义的

## Overview

关系矩阵系统是《地下排练室》的**人物关系中枢**，定义了乐队成员之间的动态关系度量、性格兼容性计算和关系变化规则。它是"人物是活的"这一支柱的核心载体——没有它，乐队成员就只是并列存在的个体，而不是一个会摩擦、会默契、会成长的群体。

**系统边界**：
- **包含**：角色配对关系值、性格兼容性计算、关系等级定义、关系变化规则、关系历史时间轴
- **不包含**：关系变化的具体事件内容（属于化学反应/叙事事件系统）、关系的视觉呈现（属于 UI 系统）、角色个体属性（属于角色属性系统）

**玩家如何感知**：
- 打开关系面板，看到林野和大伟的"默契度"从 30 涨到了 45 → 成长感
- 发现关系时间轴上多了一条记录："一起排练《锈》后，默契度 +8" → 因果感
- 看到某对角色的关系状态从"偶尔搭话"变成了"能一起喝酒" → 成就感

**为什么必须存在**：
没有关系矩阵，角色之间的互动就没有"记忆"——每次排练都是全新的，玩家看不到关系如何随时间演化。它让"化学反应"从随机事件变成了**有迹可循的积累**，是玩家产生"我在塑造这群人的关系"这一控制感的来源。

## Player Fantasy

> **"关系矩阵不是仪表盘——它是排练室墙上的那面镜子，被划花了、贴满了便签、有人在上面的灰尘里写了字又擦掉了。你每次看，都能发现新的划痕。而最深的那些，可能是你留下的。"**

你打开关系面板，看到的不是两个名字和一个数字。你看到的是一段**历史**：

```
林野 → 大伟
━━━━━━━━━━━━━━━━━━━━
Day 1:  默契度 30（"听说过这个人"）
Day 3:  +5 → 35（第一次一起排练，大伟的鼓慢了半拍，林野没骂人）
Day 7:  +8 → 43（《锈》熟练度突破40%，两人同时解锁新台词）
Day 12: -12 → 31（大伟开玩笑说林野的 solo "像猫叫"，林野三天没和他说话）
Day 15: +15 → 46（演出失败后，大伟第一个说"不是你的错"）
```

**核心情感时刻**：
你看着 Day 12 那条 -12，突然想起来——那天你让大伟选了"技巧"重点，林野精力耗尽还在硬撑。你意识到：**那条红线是我画的。** 然后你看着 Day 15 的 +15，发现它发生在一场你精心安排的"默契"排练之后。你笑了。

**参考体验**：
- 《火焰纹章》的支持度系统——但我们的关系有时间轴，不是只有当前值
- 《This Is My War》的角色关系——但我们的基调是成长而非生存

**这不是"收集全羁绊"的幻想——这是"我的选择留下了痕迹"的幻想。**

## Detailed Design

### Core Rules

**1. 关系值 (Relationship Score)**

每对角色之间有一个关系值，范围 **0–100**（整数）。

| 关系状态 | 范围 | 说明 |
|----------|------|------|
| **陌生人** | 0–19 | 几乎无互动，排练时各练各的 |
| **听说过** | 20–39 | 偶尔搭话，能一起完成基本配合 |
| **偶尔搭话** | 40–59 | 日常交流正常，排练有小默契 |
| **能一起喝酒** | 60–79 | 关系稳定，化学反应触发概率提升 |
| **知道对方怕什么** | 80–100 | 深度羁绊，可能触发专属叙事事件 |

**关系值变化规则**：
- 一起排练：+2～+8（取决于排练重点、曲目、结果）
- 触发正面化学反应：+10～+20
- 触发负面事件（吵架）：-10～-20
- 社交互动（由叙事事件系统定义）：±5～±15

**2. 性格兼容性计算**

每个角色在 4 个性格维度上有标签（来自角色属性系统）：

| 维度 | 标签 A | 标签 B |
|------|--------|--------|
| 社交 | 内向 | 外向 |
| 情感 | 敏感 | 钝感 |
| 态度 | 倔强 | 随和 |
| 反叛 | 叛逆 | 顺从 |

**兼容性计算**（针对两个角色在某一维度上的配对）：

| 配对情况 | 兼容性修正 |
|----------|-----------|
| 相反性格（如 倔强 vs 随和） | +0.20 |
| 相同性格（如 倔强 vs 倔强） | -0.20 |
| 一方有标签，另一方无对应标签 | 0.00 |
| 双方都无此维度标签 | 不参与计算 |

**总兼容性倍率**：

`compatibility_multiplier = 1.0 + Σ(dimension_bonus)`

其中 `dimension_bonus` 是每个有交互维度的修正值。

**Output Range:** 0.20（所有维度冲突）到 1.80（所有维度互补），**硬封顶至 [0.5, 1.5]**

**MVP 示例（林野 + 大伟）**：
- 林野：倔强 + 敏感
- 大伟：外向 + 随和
- 态度维度：倔强 vs 随和 = 相反 = +0.20
- 其他维度：无双方都有标签的交互 = 不参与
- `compatibility = 1.0 + 0.20 = 1.2` ✅（与放置排练系统预设一致）

**3. 关系历史时间轴**

每对角色维护一条关系变化时间轴，记录每次变化的时间、原因和数值：

| 字段 | 说明 |
|------|------|
| `timestamp` | 变化发生的游戏内时间（或天数） |
| `delta` | 变化量（+X 或 -Y） |
| `reason` | 变化原因（如"一起排练《锈》"） |
| `new_score` | 变化后的新关系值 |

**时间轴显示规则**：
- MVP 显示最近 10 条记录
- 玩家点击关系格可展开完整时间轴
- 时间轴按天聚合（同一天的多条变化合并显示）

**4. 兼容性对游戏玩法的影响**

| 影响目标 | 具体效果 |
|----------|----------|
| **化学反应触发概率** | 放置排练系统的 `compatibility_multiplier` 直接乘入触发概率 |
| **排练效率** | 关系值 ≥60 时，乐队平均排练效率 +5%；≥80 时 +10% |
| **叙事事件池** | 关系状态决定可触发的叙事事件类型（如"知道对方怕什么"解锁深度事件） |
| **演出判定** | 关系值 ≥60 时，团队配合加成 +5% |

**5. MVP 范围**

- 只实现**林野 + 大伟**这一对角色的关系
- 关系值初始为 **30**（"听说过这个人"）
- 只追踪一个维度的兼容性（态度维度：倔强 vs 随和）
- 时间轴记录最近 10 条变化

### States and Transitions

关系值本身没有离散状态——它是一个连续值。但为了方便玩家理解和事件触发，定义了 5 个关系等级：

| 等级 | 范围 | 触发事件类型 | 视觉表现 |
|------|------|-------------|----------|
| **陌生人** | 0–19 | 无 | 灰色线条，几乎不可见 |
| **听说过** | 20–39 | 基础互动事件 | 淡蓝色细线 |
| **偶尔搭话** | 40–59 | 日常交流事件 | 蓝色实线 |
| **能一起喝酒** | 60–79 | 深度交流事件、配合加成 | 亮蓝色粗线，轻微发光 |
| **知道对方怕什么** | 80–100 | 专属羁绊事件、大幅加成 | 霓虹色线条，唯一使用荧光色的关系线 |

**等级升级时**：
- 播放简短的视觉提示（线条颜色变化）
- 解锁新的可触发事件类型
- UI 显示手写风格提示："林野和大伟的关系升级了：能一起喝酒"

### Interactions with Other Systems

**输入（本系统接收的数据）**：

| 来源系统 | 输入数据 | 作用 |
|----------|----------|------|
| 角色属性系统 | 性格标签 | 计算兼容性 |
| 放置排练系统 | 排练结果（时长、重点、曲目） | 计算关系值变化 |
| 化学反应系统 | 化学反应事件结果 | 应用关系值变化 |
| 叙事事件系统 | 事件结果（和解/吵架/惊喜） | 应用关系值变化 |

**输出（本系统提供的数据）**：

| 目标系统 | 输出数据 | 作用 |
|----------|----------|------|
| 放置排练系统 | `compatibility_multiplier` | 化学反应触发概率计算 |
| 化学反应系统 | 关系状态、关系值 | 确定可触发的化学反应类型 |
| 叙事事件系统 | 关系状态、关系历史 | 确定可触发的叙事事件池 |
| 演出判定系统 | 关系值 | 团队配合加成计算 |
| UI 系统 | 关系值、时间轴、等级状态 | 关系面板展示 |

## Formulas

### 1. 性格兼容性倍率公式 (Compatibility Multiplier)

The compatibility multiplier formula is defined as:

`compatibility_multiplier = clamp(1.0 + Σ(dimension_bonus), 0.5, 1.5)`

**Variables:**
| Variable | Symbol | Type | Range | Description |
|----------|--------|------|-------|-------------|
| dimension_bonus | bonus | float | -0.2 to +0.2 | 单个维度的兼容性修正 |
| active_dimensions | count | int | 1–4 | 双方都有标签的维度数量 |

**Dimension Bonus 表：**
| 配对情况 | dimension_bonus |
|----------|----------------|
| 相反性格 | +0.20 |
| 相同性格 | -0.20 |
| 一方有标签，另一方无 | 0.00 |
| 双方都无标签 | 不参与 |

**Output Range:** 0.5 到 1.5（硬封顶）
**Example:** 林野（倔强+敏感）vs 大伟（外向+随和）
- 态度维度：倔强 vs 随和 = 相反 = +0.20
- 其他维度无双方都有标签的情况
- `compatibility = clamp(1.0 + 0.20, 0.5, 1.5) = 1.2`

### 2. 关系值变化公式 (Relationship Score Change)

The relationship score change formula is defined as:

`new_score = clamp(current_score + Σ(changes), 0, 100)`

**Variables:**
| Variable | Symbol | Type | Range | Description |
|----------|--------|------|-------|-------------|
| current_score | score | int | 0–100 | 当前关系值 |
| changes | delta | int | -20 to +20 | 各项变化的累加 |

**常见变化来源：**
| 来源 | 变化量 |
|------|--------|
| 一起排练（均衡/技巧/节省） | +2 |
| 一起排练（默契重点） | +5 |
| 触发正面化学反应 | +10～+20 |
| 触发负面事件（吵架） | -10～-20 |
| 社交互动（正面） | +5～+15 |
| 社交互动（负面） | -5～-15 |

**Output Range:** 0 到 100（硬封顶）

### 3. 排练效率团队加成公式 (Team Synergy Bonus)

The team synergy bonus formula is defined as:

`synergy_bonus = 1.0 + relationship_tier_bonus`

**Relationship Tier Bonus 表：**
| 关系等级 | 范围 | synergy_bonus |
|----------|------|---------------|
| 陌生人 | 0–19 | +0% |
| 听说过 | 20–39 | +0% |
| 偶尔搭话 | 40–59 | +0% |
| 能一起喝酒 | 60–79 | +5% |
| 知道对方怕什么 | 80–100 | +10% |

**Output Range:** 1.0 到 1.1
**Example:** 关系值 65（能一起喝酒）→ `synergy_bonus = 1.05`

## Edge Cases

- **If 关系值计算结果超过 100**：硬封顶至 100，溢出部分丢弃。
- **If 关系值计算结果低于 0**：硬保底至 0，不产生负值。
- **If 两个变化同时发生（如排练+事件）**：按顺序依次应用，先排练后事件。
- **If 关系值恰好跨越等级阈值（如从 59 到 61）**：触发等级升级提示，解锁新事件类型。
- **If 角色在关系值满 100 后继续获得正面变化**：关系值保持 100，记录变化原因到时间轴但无数值变化（"已达上限"提示）。
- **If 化学反应系统尚未实现**：使用预设关系值变化（正面化学反应=+15，负面=-15）。
- **If 某对角色从未一起排练过**：关系值保持初始值，时间轴为空。
- **If 新角色加入乐队（MVP 后）**：新角色与所有现有角色的关系初始值为 20（"听说过"）。
- **If 关系值变化导致同时跨越多个等级阈值**：只触发最高等级的升级提示。

## Dependencies

**上游依赖**：
| 系统 | 依赖数据 | 必要性 |
|------|----------|--------|
| 角色属性系统 | 性格标签 | 硬依赖 — 兼容性计算的核心输入 |

**下游被依赖**：
| 系统 | 依赖数据 | 数据流向 |
|------|----------|----------|
| 放置排练系统 | `compatibility_multiplier` | 关系矩阵 → 化学反应触发概率 |
| 化学反应系统 | 关系状态、关系值 | 关系矩阵 → 化学反应事件筛选 |
| 叙事事件系统 | 关系状态、关系历史 | 关系矩阵 → 叙事事件触发条件 |
| 演出判定系统 | 关系值 | 关系矩阵 → 团队配合加成 |

## Tuning Knobs

| 参数名 | 当前值 | 安全范围 | 过低效果 | 过高效果 | 关联系统 |
|--------|--------|----------|----------|----------|----------|
| `initial_relationship_score` | 30 | 10–50 | 角色一开始就像陌生人，关系建立太慢 | 初始关系太高，成长感不足 | — |
| `opposite_trait_bonus` | +0.20 | 0.10–0.30 | 互补性格优势不明显 | 互补性格过强，冲突性格无法使用 | 放置排练 |
| `same_trait_penalty` | -0.20 | -0.30 to -0.10 | 冲突性格惩罚太轻 | 冲突性格完全无法配合 | 放置排练 |
| `rehearsal_together_gain` | +2 | 1–5 | 一起排练几乎不涨关系 | 关系涨太快，失去积累感 | 放置排练 |
| `chemistry_positive_gain` | +15 | 5–25 | 化学反应对关系影响微弱 | 一次化学反应直接升级关系 | 化学反应 |
| `chemistry_negative_loss` | -15 | -25 to -5 | 吵架对关系影响太小 | 一次吵架关系直接崩盘 | 化学反应 |
| `synergy_bonus_tier4` | +5% | 0–10% | 高关系值无 gameplay 优势 | 玩家被迫刷关系值 | 演出判定 |
| `synergy_bonus_tier5` | +10% | 5–15% | 满关系值奖励不足 | 满关系值奖励过强 | 演出判定 |

## Visual/Audio Requirements

- **关系线**：两个角色头像之间的连线，颜色和粗细随关系等级变化（灰→蓝→亮蓝→霓虹）。
- **关系值变化动画**：数字以墨水晕染效果浮现，正向变化为蓝色，负向变化为铁锈红。
- **等级升级效果**：连线"通电"般闪烁一次，伴随一声逐渐和谐的吉他+鼓合奏（从走调到合拍）。
- **时间轴展开**：像展开一卷被胶带粘起来的排练日志，每条记录像一张撕下来的便签纸。
- **化学反应触发时的关系线**：化学反应发生时，关系线短暂变为霓虹色并振动。

## UI Requirements

- **关系矩阵面板**：2×2 网格（MVP 只有一对），每个格子显示：
  - 两个角色头像（对角排列）
  - 当前关系状态标签
  - 关系值进度条
  - 点击展开时间轴

- **时间轴详情**：
  - 纵向时间轴，最新事件在顶部
  - 每条记录显示：天数、变化量、原因
  - 正向变化用蓝色墨水，负向变化用红色墨水
  - 聚合显示：同一天的变化合并为一条

- **关系预览**：设置排练计划时，若选中的队员有关系值显示，提示"林野和大伟的默契度：43（偶尔搭话）"

## Acceptance Criteria

- **GIVEN** 林野（倔强+敏感）和大伟（外向+随和），**WHEN** 计算兼容性，**THEN** 结果为 1.2（态度维度相反+0.20）
- **GIVEN** 两个角色在态度维度都是倔强，**WHEN** 计算兼容性，**THEN** 结果为 0.8（相同性格-0.20）
- **GIVEN** 关系值当前 35，**WHEN** 一起排练一次（+2），**THEN** 新关系值为 37
- **GIVEN** 关系值当前 58，**WHEN** 触发正面化学反应（+15），**THEN** 新关系值为 73（跨越"偶尔搭话"到"能一起喝酒"）
- **GIVEN** 关系值 98，**WHEN** 获得 +10 变化，**THEN** 新关系值为 100（硬封顶），时间轴记录"已达上限"
- **GIVEN** 关系值 65（能一起喝酒等级），**WHEN** 计算排练效率加成，**THEN** 乐队平均效率 +5%
- **GIVEN** 关系值 85（知道对方怕什么等级），**WHEN** 计算排练效率加成，**THEN** 乐队平均效率 +10%
- **GIVEN** 从未一起排练过的一对角色，**WHEN** 查看时间轴，**THEN** 时间轴为空，显示"还没有共同记忆"

## Open Questions

- **Q1**: 关系值是否需要"自然衰减"机制（长时间不互动会下降），还是只增不减？当前设计为只增不减（放置游戏，进度不可逆）。
- **Q2**: MVP 只有两个角色，关系矩阵只有一对。未来扩展到 4 个角色时，矩阵会变成 4×4=16 格（含自反）。UI 如何适配？
- **Q3**: 时间轴是否需要显示"游戏内天数"还是"现实天数"？当前设计用游戏内天数。
- **Q4**: 关系状态标签的文案是否需要更个性化（如根据角色性格生成不同的标签名称）？
