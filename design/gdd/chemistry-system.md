# 化学反应系统 (Chemistry System)

> **Status**: In Design
> **Author**: [user + agents]
> **Last Updated**: 2026-04-27
> **Implements Pillar**: 选择是有意义的, 成长是可见的

## Overview

化学反应系统是《地下排练室》的**队员搭配策略层**，负责计算和触发角色之间的"化学反应"——当特定角色组合以正确方式排练时产生的额外增益、特殊效果或叙事事件。它不是独立系统，而是将"关系矩阵"和"放置排练"两个系统的数据交汇，产出策略深度：玩家不仅要考虑"谁练什么"，还要考虑"和谁一起练"。

**系统边界**：
- **包含**：化学反应类型定义、触发概率计算、效果执行、化学反应与叙事事件的连接
- **不包含**：角色基础技能成长（属于放置排练系统）、角色关系值存储（属于关系矩阵系统）、叙事事件具体内容（属于叙事事件系统）、演出时的配合技效果（属于内容数据库/演出判定系统）

**玩家如何感知**：
- 林野和大伟一起排练《锈》，触发了"倔强节奏" → 鼓和吉他的配合度临时提升 → 演出时更容易获得好评
- 选择"默契"排练重点 → 化学反应触发概率翻倍 → "他们在培养默契"
- 长期只让林野一个人排练 →  never triggers chemistry → "这支乐队各玩各的"

**为什么必须存在**：
没有化学反应系统，乐队只是一群独立训练的角色。化学反应让"乐队"这个词有了意义——1+1 > 2。它是"选择是有意义的"支柱的深度来源：选谁一起排练、选什么重点、选哪首歌，都会影响化学反应。

## Player Fantasy

> **"好的乐队不是把最好的乐手聚在一起——是把对的人放在一起。"**

你让林野和大伟一起排练《锈》。第三次排练时，屏幕角落闪过一个提示："倔强节奏触发——大伟的鼓点跟上了林野那段倔强的吉他 riff。"

你不知道这是什么意思。但下一次演出时，当《锈》演到那段 riff，你注意到观众中有个人抬起了头——之前他一直在看手机。演出结束后，那条声誉比平时多了 3 点。

你回看排练记录，发现每次林野和大伟一起练《锈》都有概率触发这个效果。你开始尝试：如果选"默契"重点呢？触发概率翻倍。如果他们的关系值再高一点呢？效果更强。

你意识到：**这不是运气，这是策略。** 你在学这支乐队的"化学反应"。

**核心情感时刻**：
当你第一次故意安排林野和大伟连续排练同一首歌，第三次终于触发了高级版"倔强节奏"，屏幕上出现了两个人对视一眼的小动画——没有对话，但你知道这意味着什么。那一刻你不是在优化数值，你是在**见证两个人的默契诞生**。

**参考体验**：
- 《火焰纹章》的支持对话——角色并肩作战产生羁绊，羁绊带来战斗加成（但我们的化学反应更隐性，不强制玩家刷）
- 《女神异闻录5》的协力技——特定组合在特定条件下触发特殊效果

**这不是"最强阵容"的幻想——这是"他们发现彼此"的幻想。**

## Detailed Design

### Core Rules

#### 1. 化学反应类型 (Chemistry Types)

化学反应分为 3 个层级：

| 层级 | 名称 | 触发条件 | 效果强度 | MVP 数量 |
|------|------|---------|---------|---------|
| **基础反应** | 日常默契 | 关系值 ≥ 40，任意排练 | 小幅增益 | 每对角色 1 个 |
| **核心反应** | 专属配合 | 关系值 ≥ 60，特定曲目 | 中幅增益 + 叙事 | 每对角色 1 个 |
| **巅峰反应** | 传奇时刻 | 关系值 ≥ 80，特定条件 | 大幅增益 + 专属事件 | MVP 1 个 |

**MVP 范围**：只实现林野+大伟的 1 个基础反应 + 1 个核心反应。其他角色组合 Post-MVP 扩展。

#### 2. MVP 化学反应定义

**基础反应：日常同步 (Daily Sync)**
- **触发条件**：林野和大伟一起排练，关系值 ≥ 40
- **触发时机**：排练结束时判定
- **效果**：下次演出中，两人的乐器技能临时 +5%
- **持续时间**：1 次演出
- **UI 提示**：小型浮动文字 "默契+"

**核心反应：倔强节奏 (Stubborn Rhythm)**
- **触发条件**：
  - 林野 + 大伟一起排练
  - 关系值 ≥ 60
  - 排练曲目为《锈》或《地下室之光》
  - 排练重点为"默契"或"技巧"
- **触发时机**：排练结束时判定
- **效果**：
  - 下次演出中，鼓+吉他的配合度 +15%
  - 如果演出曲目与排练曲目相同：额外 +10% 熟练度收益
  - 触发后关系值 +5
- **UI 提示**：特殊动画（鼓点和吉他 riff 的视觉同步）+ 文字 "倔强节奏！"
- **叙事连接**：触发后 50% 概率触发叙事事件 "那段 riff 之后"（两人关于音乐理念的对话）

#### 3. 触发概率公式

```
chemistry_chance = base_chance × compatibility_multiplier × focus_multiplier × relationship_multiplier × song_multiplier × streak_multiplier
```

**Base Chance (基础概率)**：
- 基础反应：20%
- 核心反应：15%
- 巅峰反应：10%

**Compatibility Multiplier (兼容性倍率)**：
- 来自关系矩阵系统
- 林野+大伟：1.2（倔强 vs 随和互补，敏感 vs 钝感互补）
- 范围：0.5（冲突）到 1.5（完美互补）

**Focus Multiplier (排练重点倍率)**：
- 均衡：1.0
- 技巧：1.0
- 默契：2.0（翻倍）
- 节省：0.5（减半）

**Relationship Multiplier (关系值倍率)**：
```
relationship_multiplier = 0.5 + (relationship_score / 100) × 0.5
```
- 关系值 0：0.5×
- 关系值 50：0.75×
- 关系值 100：1.0×

**Song Multiplier (曲目倍率)**：
- 核心反应要求特定曲目：匹配曲目时 1.5×，不匹配时 0（无法触发）
- 基础反应无曲目要求：1.0

**Streak Multiplier (连击倍率)**：
- 连续多次排练同一曲目且同一组合：
  - 第 1 次：1.0×
  - 第 2 次：1.2×
  - 第 3 次：1.4×
  - 第 4 次+：1.5×（封顶）
- 更换曲目或组合后重置

#### 4. 化学反应效果执行

**效果类型**：

| 效果类型 | 说明 | 示例 |
|---------|------|------|
| **技能增益** | 临时提升特定乐器技能 | 吉他/鼓技能 +5% |
| **熟练度增益** | 提升曲目熟练度成长速度 | 熟练度成长 ×1.2 |
| **关系值增益** | 增加角色关系值 | 关系值 +5 |
| **状态恢复** | 恢复角色状态值 | 士气 +10 |
| **演出加成** | 演出时获得额外加成 | 配合度 +15% |
| **叙事触发** | 触发特定叙事事件 | 事件 "那段 riff 之后" |

**效果叠加规则**：
- 同类型效果：取最高值，不叠加（避免数值爆炸）
- 不同类型效果：可以同时存在
- 效果持续时间：都是"直到下次演出"或"本次排练期间"

#### 5. 化学反应记录

系统记录最近触发的化学反应：

```json
{
  "chemistry_history": [
    {
      "timestamp": 1713800000,
      "type": "stubborn_rhythm",
      "members": ["linye", "dawei"],
      "song": "rust",
      "focus": "chemistry",
      "effects": ["演出配合度+15%", "熟练度+10%"]
    }
  ]
}
```

#### 6. 化学反应与演出配合技的区别

| | 化学反应 | 演出配合技 |
|--|---------|-----------|
| **触发时机** | 排练结束时 | 演出进行中 |
| **持续时间** | 持续到下次演出 | 本次演出内 |
| **效果** | 为下次演出做准备 | 直接改变演出判定 |
| **数据来源** | 关系矩阵 + 排练历史 | 内容数据库 |
| **玩家感知** | "他们练出了默契" | "他们现场爆发了" |

**联动关系**：化学反应触发的效果 → 提升演出时的基础能力 → 演出配合技更容易触发 → 更好的演出结果。

### States and Transitions

化学反应系统本身是无状态的触发器，但在游戏中有以下阶段：

| 阶段 | 触发条件 | 系统行为 |
|------|---------|---------|
| **潜在** | 玩家设置排练计划 | 系统计算可能触发的反应类型和概率 |
| **激活** | 排练结束 | 执行触发判定，如果成功则应用效果 |
| **生效** | 效果应用后 | 效果持续到条件过期（如下次演出后） |
| **消退** | 演出完成或时间过期 | 临时效果清除 |
| **记录** | 每次触发 | 写入历史记录 |

## Formulas

### 1. 化学反应触发概率公式 (Chemistry Trigger Chance)

```
chemistry_chance = base_chance × compatibility_multiplier × focus_multiplier × relationship_multiplier × song_multiplier × streak_multiplier
```

**Variables:**
| Variable | Symbol | Type | Range | Description |
|----------|--------|------|-------|-------------|
| base_chance | bc | float | 0.10-0.20 | 反应层级的基础概率 |
| compatibility_multiplier | cm | float | 0.5-1.5 | 性格兼容性倍率（来自关系矩阵系统） |
| focus_multiplier | fm | float | 0.5-2.0 | 排练重点影响 |
| relationship_multiplier | rm | float | 0.5-1.0 | 关系值影响 |
| song_multiplier | sm | float | 0.0-1.5 | 特定曲目匹配 |
| streak_multiplier | stm | float | 1.0-1.5 | 连续同曲目排练加成 |

**Sub-formulas:**

**Relationship Multiplier:**
```
relationship_multiplier = 0.5 + (relationship_score / 100) × 0.5
```

**Streak Multiplier:**
```
streak_multiplier = min(1.5, 1.0 + (consecutive_count - 1) × 0.2)
```

**Output Range:** 0% to 67.5% (0.20 × 1.5 × 2.0 × 1.0 × 1.5 × 1.5 = 0.675)

**Example (倔强节奏):**
- base = 0.15
- compatibility = 1.2
- focus = 2.0 (默契重点)
- relationship = 0.65 (关系值 30)
- song = 1.5 (匹配曲目)
- streak = 1.2 (第2次连续)
- `chance = 0.15 × 1.2 × 2.0 × 0.65 × 1.5 × 1.2 = 0.4212 = 42.1%`

---

### 2. 效果强度公式 (Effect Strength)

效果强度由关系值和反应层级决定：

```
effect_strength = base_strength × (0.5 + relationship_score / 100 × 0.5) × tier_multiplier
```

**Tier Multiplier:**
| 层级 | Multiplier |
|------|-----------|
| 基础反应 | 1.0 |
| 核心反应 | 1.5 |
| 巅峰反应 | 2.0 |

**Output Range:**  varies by effect type
**Example:** 核心反应效果 base=15%, 关系值 70, tier=1.5
→ `strength = 15% × (0.5 + 0.7 × 0.5) × 1.5 = 15% × 0.85 × 1.5 = 19.1%`

---

### 3. 连击计数器公式 (Streak Counter)

```
consecutive_count = current_consecutive_count + 1  if (same_members AND same_song)
                    = 1                              otherwise
```

**Reset Conditions:**
- 更换排练曲目
- 更换参与角色
- 跳过本次排练（角色精力不足导致部分退出）
- 超过 48 小时未排练（离线时间太长，默契感消退）

**Output Range:** 1 to ∞ (but multiplier caps at 5th consecutive)

---

### 4. 化学反应收益公式 (Chemistry Value)

衡量一次化学反应的"价值"（用于平衡分析）：

```
chemistry_value = effect_strength × duration × rarity_weight
```

**Rarity Weight:**
| 反应层级 | Weight |
|---------|--------|
| 基础反应 | 1.0 |
| 核心反应 | 2.5 |
| 巅峰反应 | 5.0 |

**Output Range:** arbitrary units for balancing
**Example:** 核心反应效果强度 19.1%, 持续 1 次演出, rarity=2.5
→ `value = 19.1 × 1 × 2.5 = 47.8`

## Edge Cases

- **If 关系值低于反应要求的阈值**: 无法触发该层级反应。基础反应要求 ≥40，核心反应要求 ≥60。即使其他条件完美，关系值不足 = 不触发。
- **If 排练中途角色退出（精力耗尽）**: 连击计数器重置。化学反应只在"完整排练"（所有初始角色坚持到最后）后判定。
- **If 多个化学反应同时满足条件**: 只触发最高层级的一个。例如同时满足基础反应和核心反应条件 → 触发核心反应，不重复触发基础反应。
- **If 效果类型相同**: 新效果覆盖旧效果，取较强者。例如上次触发了 +5% 技能，这次触发了 +8% → 保留 +8%。
- **If 48 小时未排练**: 连击计数器重置。默契感会随时间消退。
- **If 化学反应触发但演出取消**: 效果持续到演出真正发生，或 7 天后过期（防止无限累积）。
- **If 内容数据库中未定义某对角色的化学反应**: 无法触发任何反应。MVP 只定义林野+大伟。
- **If 关系值因负面事件暴跌**: 已激活的效果不受影响（效果在触发时已锁定），但未来触发概率降低。
- **If 玩家从未触发过化学反应**: 游戏不会提示"化学反应系统存在"。玩家自然发现——第一次触发时弹出教程式解释。

## Dependencies

### 输入依赖（本系统需要的数据）

| 来源系统 | 输入数据 | 作用 |
|----------|----------|------|
| 关系矩阵系统 | 关系值、兼容性倍率 | 触发概率计算、效果强度 |
| 放置排练系统 | 排练计划（成员、曲目、重点）、排练结果 | 触发条件判定 |
| 角色属性系统 | 角色状态值 | mood_bonus 计算 |
| 内容数据库 | 化学反应定义（哪些角色组合有什么反应） | 反应类型查找 |

### 输出依赖（使用本系统数据的系统）

| 目标系统 | 输出数据 | 作用 |
|----------|----------|------|
| 放置排练系统 | 化学反应效果（技能增益、熟练度增益等） | 修改排练收益 |
| 演出判定系统 | 演出加成效果 | 影响演出评分 |
| 叙事事件系统 | 化学反应触发信号 | 触发相关叙事事件 |
| 关系矩阵系统 | 关系值变化（某些反应增加关系值） | 反馈循环 |
| 声誉系统 | 配合技触发额外声誉 | 声誉奖励 |

### 双向依赖说明

- **关系矩阵系统**：关系值影响触发概率 → 触发后某些反应增加关系值。正反馈循环，但关系值本身主要由排练积累，化学反应是加速器而非主要来源。
- **放置排练系统**：排练产生触发条件 → 化学反应修改排练收益。双向但清晰：排练是"因"，化学反应是"果"，效果修改是"反馈"。

### 依赖顺序

```
角色属性系统 ──┐
关系矩阵系统 ──┼──→ 化学反应系统 ──→ 演出判定系统
放置排练系统 ──┤      ↓
内容数据库 ────┘   叙事事件系统
```

## Tuning Knobs

| 参数 | 当前值 | 安全范围 | 调整影响 | 所在公式 |
|------|--------|---------|---------|---------|
| `base_chance_basic` | 0.20 | 0.15-0.30 | 基础反应触发频率。越高=日常默契越常见 | `base_chance` |
| `base_chance_core` | 0.15 | 0.10-0.25 | 核心反应触发频率。越高=专属配合越常见 | `base_chance` |
| `base_chance_peak` | 0.10 | 0.05-0.20 | 巅峰反应触发频率。越高=传奇时刻越常见 | `base_chance` |
| `focus_chemistry_multiplier` | 2.0 | 1.5-3.0 | "默契"重点对触发概率的加成。越高=默契重点越有价值 | `focus_multiplier` |
| `streak_increment` | 0.2 | 0.1-0.3 | 每次连续排练的触发概率增幅。越高=连击策略越强 | `streak_multiplier` |
| `streak_cap` | 1.5 | 1.3-2.0 | 连击倍率上限。越高=长期连击收益越大 | `streak_multiplier` |
| `relationship_threshold_basic` | 40 | 20-50 | 基础反应的关系值门槛。越低=化学反应越早出现 | 触发条件 |
| `relationship_threshold_core` | 60 | 50-70 | 核心反应的关系值门槛。越低=深度配合越早出现 | 触发条件 |
| `streak_reset_hours` | 48 | 24-72 | 连击计数器重置的离线时间。越短=越需要频繁排练 | 连击重置 |
| `effect_duration_base` | 1 | 1-3 | 基础反应效果持续演出次数。越长=效果越持久 | 效果系统 |

**调优策略**：
- 如果玩家反馈"化学反应太稀有，感觉不到存在"：提高 `base_chance_core` 或降低 `relationship_threshold_core`
- 如果玩家反馈"化学反应太强，变成唯一策略"：降低 `focus_chemistry_multiplier` 或 `streak_cap`
- 如果玩家从不选"默契"重点：提高 `focus_chemistry_multiplier` 或给默契重点增加其他收益

## Visual/Audio Requirements

### 视觉需求

**基础反应触发提示**：
- 小型浮动文字，从排练界面底部升起
- 文字："日常同步" + 具体效果（如 "下次演出 +5%"）
- 颜色：柔和的蓝白色
- 持续时间：2 秒

**核心反应触发提示**：
- 屏幕中央出现角色头像的对视动画（0.5 秒）
- 头像之间有微弱的连线效果（颜色代表反应类型）
- 文字：反应名称 + 效果列表
- 持续时间：3 秒
- 背景轻微模糊（突出提示）

**巅峰反应触发提示**：
- 全屏特效（类似垃圾摇滚海报的风格）
- 角色立绘剪影 + 反应名称大字
- 屏幕边框有短暂的"震动"效果
- 持续时间：4 秒
- 玩家可点击跳过

**效果激活状态指示器**：
- 主界面角色卡片上有小型图标表示"化学反应效果激活中"
- hover 显示具体效果内容和剩余次数

### 音频需求

**基础反应音效**：
- 两个音符和谐共鸣（吉他 + 鼓的简短和弦）

**核心反应音效**：
- 一段简短的鼓点 + 吉他 riff（与反应名称呼应）
- 如"倔强节奏"：倔强的吉他失真音色 + 坚定的鼓点

**巅峰反应音效**：
- 完整的 3-4 秒音乐片段
- 乐队全奏的高潮段落
- 伴随观众欢呼声（短）

**效果到期音效**：
- 轻微的"消退"音效，提示化学反应效果已过期

## UI Requirements

### 排练界面

**化学反应预览**：
- 位置：排练计划确认前，在"预计收益"区域下方
- 内容：
  - 如果当前组合有已知的化学反应：显示可能触发的反应名称 + 触发概率
  - 例如："倔强节奏 — 预计触发概率 35%"
  - 如果关系值不足：显示灰色文字 "关系值不足（需要 60+）"
  - 如果该组合无化学反应定义：显示 "无已知化学反应"

**化学反应效果追踪**：
- 在角色状态卡片的角落显示小图标
- 图标数量 = 当前激活的效果数量
- 点击展开详情面板

### 化学反应详情面板

**面板内容**：
- 当前激活的所有化学反应效果列表
- 每个效果显示：名称、来源（哪次排练）、剩余次数、具体数值
- 历史记录：最近 10 次触发的化学反应
- 筛选：按角色组合 / 按曲目 / 按反应层级

### 演出结果界面

**化学反应贡献显示**：
- 如果本次演出受益于化学反应：单独列出
- 显示："得益于倔强节奏，配合度 +15%"
- 如果化学反应触发了配合技：显示配合技名称和额外收益

### 角色详情界面

**化学反应标签页**：
- 显示该角色与其他所有角色的化学反应状态
- 已解锁的反应：显示名称、触发次数、效果说明
- 未解锁的反应：显示灰色轮廓 + 解锁条件提示（如 "关系值达到 60 解锁"）
- 进度条：显示距离下一个反应层级还差多少关系值

## Acceptance Criteria

### 功能测试

- [ ] 林野+大伟排练《锈》+默契重点 → 可能触发"倔强节奏"
- [ ] 关系值 < 60 时 → 无法触发核心反应
- [ ] 关系值 ≥ 60 但曲目不匹配 → 无法触发"倔强节奏"（可触发基础反应）
- [ ] 连续排练同一曲目 3 次 → streak_multiplier = 1.4
- [ ] 更换曲目后 → streak 重置为 1
- [ ] 触发"倔强节奏" → 下次演出时配合度 +15%
- [ ] 同一类型效果不叠加 → 取最高值
- [ ] 演出后 → 临时效果清除

### 边界测试

- [ ] 48 小时未排练 → streak 重置
- [ ] 排练中途角色退出 → streak 重置
- [ ] 多个反应同时满足 → 只触发最高层级
- [ ] 演出取消 → 效果 7 天后过期
- [ ] 关系值因事件暴跌 → 已激活效果不受影响
- [ ] 内容数据库无定义组合 → 不触发任何反应

### 集成测试

- [ ] 从关系矩阵系统读取关系值和兼容性 → 正确计算概率
- [ ] 从放置排练系统接收排练结果 → 正确执行触发判定
- [ ] 输出效果到放置排练系统 → 正确修改排练收益
- [ ] 输出效果到演出判定系统 → 正确影响演出评分
- [ ] 触发叙事事件 → 正确通知叙事事件系统
- [ ] 声誉奖励 → 正确通知声誉系统

### 性能测试

- [ ] 触发判定计算耗时 < 1ms
- [ ] 历史记录查询（20 条）耗时 < 5ms

## Open Questions

- **三人或四人排练时，化学反应如何计算？** MVP 只有 2 个角色（林野+大伟），但未来 3-4 人时，是检查所有两两组合，还是只检查"核心配对"？
- **化学反应的叙事包装需要多少内容？** MVP 只实现 1-2 个反应，但每个反应是否需要专属对话/事件文本？
- **巅峰反应是否在 MVP 实现？** 当前设计包含 3 个层级，但 MVP 只实现基础+核心。巅峰反应是否完全留到 Post-MVP？
- **化学反应是否可以在演出中临时触发？** 当前设计是"排练时触发，演出时生效"。是否允许演出过程中随机触发（如"即兴配合"）？
- **负面化学反应是否存在？** 例如关系值极低时触发"配合失误"，降低演出表现？当前设计只有正面效果，但负面效果可能增加策略深度。
- **化学反应的发现机制：** 玩家是否一开始就知道所有可能的化学反应？还是需要通过尝试来"发现"？发现后是否永久显示在 UI 中？
