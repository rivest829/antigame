# 叙事事件UI (Narrative Event UI)

> **Status**: Designed (pending review)
> **Author**: [user + agents]
> **Last Updated**: 2026-04-29
> **Implements Pillar**: 人物是活的

## Overview

叙事事件UI是《地下排练室》的**故事体验窗口**——负责将叙事事件系统推送的事件数据转化为玩家可感知、可交互的叙事体验。如果说叙事事件系统是"导演"，叙事事件UI就是"银幕+放映机"——它决定了玩家如何看到故事、如何参与故事、如何感受故事的分量。

**系统边界**：
- **包含**：事件弹窗界面、事件卡片设计、分支选项展示、打字机文本效果、选择反馈动画、事件历史列表
- **不包含**：事件的触发逻辑（属于叙事事件系统）、模板定义（属于事件模板系统）、后果的具体应用（属于各业务系统）

**玩家如何感知**：
- 屏幕上弹出一张皱巴巴的纸条 → "有人在给我留言"
- 文字一个字一个字出现 → "我在阅读一个故事"
- 分支选项像用马克笔写的 → "我需要做选择"
- 选择后纸条烧掉 → "这个选择不可逆"

**为什么必须存在**：
没有叙事事件UI，叙事事件系统只是一堆后台数据——事件触发了、后果应用了，但玩家完全无感知。叙事事件UI是把"后台变化"翻译成"前端体验"的关键层。它是"人物是活的"这一支柱的最终呈现：玩家不是在看数据变化，而是在读一封信、做一道选择题、感受一个后果。

## Player Fantasy

> **屏幕上弹出一张被揉皱的A4纸。**
>
> 纸的边缘有撕裂的痕迹，像是从某个本子上撕下来的。纸的右上角有一块咖啡渍——你认出那是大伟经常喝的廉价速溶咖啡的颜色。
>
> 纸上是大伟潦草的字迹：
>
> "林野今天又把《锈》的solo改了。我说原来的版本更有感觉，他说我根本不懂音乐。"
>
> "我们吵了20分钟。"
>
> "我现在不想说话。"
>
> 文字是一个字一个字出现的，像有人在实时写给你看。每个字出现时都有轻微的"沙沙"声，像铅笔在纸上摩擦。
>
> 文字完全显示后，纸的下方出现了三个选项：
> 1. [支持林野：创新是乐队的灵魂]
> 2. [支持大伟：经典版本确实更有味道]
> 3. [各打五十大板：你们俩都冷静一下]
>
> 每个选项都是手写的，像用马克笔写在胶带上的。选项2旁边有一滴墨水晕开了——你猜那是大伟写的。
>
> 你点了选项3。
>
> 纸条开始从边缘燃烧，像被烟头点着了。燃烧的速度不快，你有足够的时间看完整张纸变成灰烬。灰烬飘落，消失在屏幕底部。
>
> 然后出现了一行淡灰色的字：
> "两天后，林野主动找大伟道歉了。"
>
> 这行字下面，有两个数字在浮动：
> - 林野心情 -5
> - 大伟心情 -5
> - 林野-大伟关系值 不变
>
> 数字浮动3秒后，屏幕恢复了主界面。右上角的未读事件数从2变成了1。
>
> **核心情感时刻**：当你发现选项2（支持大伟）被锁定，hover显示"大伟今天不想被支持"时，你意识到——**这个选项的不可用状态本身就是叙事。** 大伟不是因为系统bug而不能选，而是因为"他现在不想说话"。

**这不是"读对话"的幻想——这是"收到一封信"的幻想。** 你打开的不是一个UI面板，你打开的是某人留在你桌上的纸条。

## Detailed Design

### Core Rules

#### 1. 事件弹窗整体结构

事件弹窗采用全屏覆盖式设计：

```
┌─────────────────────────────────────────┐
│ [半透明黑色遮罩]                         │
│                                         │
│      ┌─────────────────────────┐        │
│      │ [事件类型图标]           │        │
│      │                         │        │
│      │ 标题：林野和大伟的争执   │        │
│      │ ─────────────────────── │        │
│      │                         │        │
│      │ 正文（打字机效果）        │        │
│      │                         │        │
│      │ ─────────────────────── │        │
│      │                         │        │
│      │ [选项1] [选项2] [选项3]  │        │
│      │                         │        │
│      │ [时间戳]                │        │
│      └─────────────────────────┘        │
│                                         │
└─────────────────────────────────────────┘
```

**事件卡片样式**：
- 背景：泛黄的纸张纹理，边缘有撕裂效果
- 边框：不规则，像手工撕纸
- 阴影：轻微投影，营造"悬浮在桌面上"的感觉
- 类型图标：根据事件类型显示不同图标（RELATIONSHIP=两个人形剪影，REHEARSAL=乐器轮廓）

#### 2. 打字机文本效果

**文字显示规则**：
- 文字逐个字符出现
- 速度：默认25字符/秒（可在设置中调整15-40）
- 标点符号停顿：句号/逗号额外停顿200ms，感叹号/问号额外停顿400ms
- 玩家点击屏幕：立即显示完整文本

**文字样式**：
- 标题：大号手写体，加粗
- 正文：中等手写体，行间距1.5
- 变量填充内容（如角色名）：颜色与角色主题色一致
- 情感词：轻微放大或变色

#### 3. 分支选项展示

**选项按钮设计**：
- 背景：胶带纹理（像用胶带贴在纸上的纸条）
- 文字：手写体，黑色
- 边框：无（胶带边缘不规则）
- 间距：选项之间留足空间（避免误触）

**选项状态**：

| 状态 | 视觉表现 | 交互 |
|------|---------|------|
| **可用** | 正常胶带，手写文字，可点击 | 点击后进入选择处理 |
| **不可用** | 灰色胶带，文字模糊，有斜线 | 不可点击，hover显示原因 |
| **已选中** | 胶带被"撕下"，留下空白痕迹 | 不可再次点击 |

**选项布局**：
- 1个选项：居中显示
- 2个选项：左右并排
- 3个选项：上中下纵向排列（或根据内容长度自适应）

#### 4. 选择反馈动画

**选择处理流程**：
```
玩家点击选项
  → 选项按钮被"撕下"（撕纸动画，300ms）
  → 其他选项按钮淡出（200ms）
  → 事件卡片开始燃烧（从边缘向中心，500ms）
  → 燃烧完成后，显示后果文字（淡入，300ms）
  → 后果数字浮动（300ms）
  → 全部显示3秒后，自动关闭或点击关闭
```

**后果显示**：
- 文字后果：淡入显示，如"两天后，林野主动找大伟道歉了"
- 数值后果：浮动数字，如"林野心情 -5"，带颜色（红色=减少，绿色=增加）
- 连锁事件：如果有连锁事件，显示"新事件将在X小时后触发"

#### 5. 纯叙事事件（无分支）

部分事件没有分支选项（如MILESTONE类型的纯叙事）：

**展示流程**：
```
打字机显示正文
  → 显示"继续"按钮（胶带风格）
  → 玩家点击
  → 事件卡片燃烧消失
  → 关闭弹窗
```

**"继续"按钮**：
- 样式与分支选项一致
- 位置：卡片底部居中
- 点击后无后果显示（因为没有选择）

#### 6. 事件历史列表

**入口**：主界面 → 角色详情UI → "事件历史"标签页

**列表项设计**：
```
┌─────────────────────────────────────────┐
│ [类型图标] 林野和大伟的争执              │
│ 关系事件 · 2026-04-15 14:30             │
│ 你的选择：各打五十大板                   │
│ 后果：关系值不变                         │
└─────────────────────────────────────────┘
```

**列表功能**：
- 按时间倒序排列
- 支持筛选：按类型 / 按角色 / 按时间段
- 点击某条：展开查看完整内容（包括未选中的分支选项，灰色显示）
- 最大显示：50条（超出部分可加载更多）

#### 7. 连锁事件连续展示

当选择后果触发连锁事件时：

**展示流程**：
```
当前事件处理完成
  → 显示"新事件发生！"提示（1秒）
  → 新事件卡片从右侧滑入
  → 旧事件卡片从左侧滑出
  → 新事件的打字机效果开始
```

**过渡动画**：
- 卡片滑动：300ms，ease-out
- 两张卡片短暂重叠：100ms
- 营造"连续叙事"的感觉

### States and Transitions

叙事事件UI作为展示层，有以下显示状态：

| 状态 | 说明 | 转换条件 |
|------|------|---------|
| **文本展示 (Text_Display)** | 打字机效果播放中 | 进入事件时 |
| **等待选择 (Awaiting_Choice)** | 文本播放完毕，等待玩家选择 | 打字机完成后 |
| **选择反馈 (Choice_Feedback)** | 玩家已选择，展示后果 | 点击选项后 |
| **关闭过渡 (Closing)** | 后果展示完毕，关闭弹窗 | 自动或点击关闭 |

```
进入事件 → 文本展示 → 等待选择 → 选择反馈 → 关闭过渡
                              ↓（纯叙事事件无分支）
                         点击"继续" → 关闭过渡
```

### Interactions with Other Systems

**输入（本系统需要的数据）**：

| 来源系统 | 输入数据 | 作用 |
|----------|----------|------|
| 叙事事件系统 | 事件实例数据（标题/正文/分支/状态） | UI展示内容 |
| 事件模板系统 | 文本模板（变量已填充） | 展示文本 |
| 角色属性系统 | 角色主题色 | 变量文字着色 |

**输出（本系统提供的数据）**：

| 目标系统 | 输出数据 | 作用 |
|----------|----------|------|
| 叙事事件系统 | 玩家选择的分支ID | 处理选择后果 |
| 主界面UI | 事件已读/已处理信号 | 更新红点 |

## Formulas

### 1. 打字机显示进度 (Typewriter Progress)

```
visible_chars = min(total_chars, floor(elapsed_time × chars_per_second))
```

**Variables:**
| Variable | Symbol | Type | Range | Description |
|----------|--------|------|-------|-------------|
| chars_per_second | cps | float | 15-40 | 打字速度 |
| elapsed_time | t | float | 0-∞ | 已过去的时间 |
| total_chars | total | int | 1-∞ | 文本总字符数 |
| pause_period | pause | float | 0.2-0.4s | 标点停顿 |

**Example**: 文本100字符，cps=25，无标点
- t=0: visible=0
- t=2: visible=50
- t=4: visible=100（完成）

---

### 2. 燃烧动画进度 (Burn Animation Progress)

```
burn_radius = max_radius × ease_in_out(t)
where t = elapsed_time / burn_duration
```

**Variables:**
| Variable | Symbol | Type | Range | Description |
|----------|--------|------|-------|-------------|
| burn_duration | dur | float | 400-800ms | 燃烧动画时长 |
| max_radius | radius | float | - | 卡片对角线的一半 |

## Edge Cases

- **If 事件文本过长（超过500字符）**：支持滚动阅读，底部选项按钮始终固定在屏幕下方可见。
- **If 玩家在打字机效果播放期间点击屏幕**：立即显示完整文本，并进入"等待选择"状态。
- **If 玩家在打字机播放期间切换应用到后台**：暂停打字机，返回前台时继续播放。
- **If 连锁事件连续触发超过3个**：显示提示"还有更多事件……"，防止玩家疲劳。玩家可选择"继续处理"或"稍后查看"。
- **If 所有分支选项均不可用**：显示"继续"按钮，事件无选择直接推进。记录日志供调试。
- **If 事件卡片正在燃烧时玩家点击屏幕**：无响应，等待燃烧动画完成。
- **If 事件UI正在展示时收到新事件**：新事件加入队列，当前事件处理完成后自动展示下一个。
- **If 事件历史记录被删除**：历史列表显示"没有历史记录"空状态。

## Tuning Knobs

| 参数 | 当前值 | 安全范围 | 调整影响 |
|------|--------|---------|---------|
| `typewriter_speed_cps` | 25 | 15-40 | 打字机速度。越快=阅读越快，但可能缺乏沉浸感 |
| `typewriter_pause_period_ms` | 200 | 100-400 | 标点停顿时长。越长=叙事节奏越慢 |
| `burn_animation_duration_ms` | 600 | 400-1000 | 燃烧动画时长。越长=后果前的仪式感越强 |
| `card_enter_duration_ms` | 400 | 300-600 | 卡片进入动画时长 |
| `consequence_display_duration_ms` | 3000 | 2000-5000 | 后果文字显示时长。越长=玩家越能消化后果 |
| `chain_event_threshold` | 3 | 2-5 | 连续事件疲劳阈值。超过后提示休息 |
| `text_max_length_scroll` | 500 | 300-800 | 触发滚动显示的最小字符数 |
| `history_entries_per_page` | 10 | 5-20 | 事件历史每页显示条数 |

## Visual/Audio Requirements

### 视觉需求

**事件卡片**：
- 背景：泛黄纸张纹理，分辨率≥512×512
- 边缘：不规则撕裂，随机微卷角
- 咖啡渍/墨水渍：随机分布在卡片边缘（不同事件不同位置）
- 类型图标：手绘风格，位置在卡片左上角

**打字机效果**：
- 每个字符出现时：轻微"墨水晕染"效果（从中心向外扩散，50ms）
- 光标：闪烁的竖线（|），文字显示完成后消失
- 变量文字（角色名）：使用角色主题色，轻微加粗

**燃烧动画**：
- 从卡片边缘开始，像被烟头点燃
- 燃烧区域：黑色→橙色→灰色的渐变
- 燃烧速度：不均匀（某些角燃烧更快）
- 燃烧完成后：留下黑色灰烬轮廓，然后消散

**分支选项**：
- 胶带纹理：半透明黄色，边缘不规则
- 文字：黑色马克笔手写体
- hover效果：胶带轻微翘起（3D倾斜）
- 点击效果：胶带被"撕下"，留下纸面上的撕痕

**后果数字浮动**：
- 数字从事件卡片中央浮出
- 向上漂移50px，同时透明度从1→0
- 持续时间：1.5秒
- 颜色：绿色（增加）/ 红色（减少）/ 白色（不变）

### 音频需求

**打字机音效**：
- 每个字符：轻微的"沙沙"声（铅笔在纸上摩擦）
- 音量：极低（不干扰阅读）
- 标点符号：额外的轻微停顿声

**燃烧音效**：
- 纸张燃烧的"噼啪"声（轻微）
- 音量：中等

**胶带撕下**：
- "嘶"的一声（像撕开胶带）
- 短促，200ms

**后果数字浮动**：
- 轻微的"叮"声（增加）或"咚"声（减少）
- 每个数字独立发声，间隔200ms

## Acceptance Criteria

### 功能测试

- **GIVEN** 事件文本100字符，**WHEN** 进入事件弹窗，**THEN** 打字机效果正确播放，约4秒完成
- **GIVEN** 打字机播放中，**WHEN** 点击屏幕，**THEN** 文本立即完全显示
- **GIVEN** 事件有3个分支选项，**WHEN** 文本播放完成，**THEN** 3个选项正确显示
- **GIVEN** 选项1条件不满足，**WHEN** 展示选项，**THEN** 选项1灰色不可点，hover显示原因
- **GIVEN** 点击可用选项，**WHEN** 选择后，**THEN** 卡片燃烧动画播放，后果正确显示
- **GIVEN** 纯叙事事件无分支，**WHEN** 文本播放完成，**THEN** 显示"继续"按钮

### 边界测试

- **GIVEN** 事件文本600字符，**WHEN** 进入弹窗，**THEN** 文本区域支持滚动，选项始终可见
- **GIVEN** 连锁事件连续触发4个，**WHEN** 处理第3个后，**THEN** 显示"还有更多事件……"提示
- **GIVEN** 事件UI展示中切换应用到后台，**WHEN** 返回前台，**THEN** 打字机从暂停处继续
- **GIVEN** 所有分支不可用，**WHEN** 进入事件，**THEN** 显示"继续"按钮，无选择直接推进

### 集成测试

- **GIVEN** 从叙事事件系统读取事件数据，**WHEN** 展示弹窗，**THEN** 标题/正文/分支正确显示
- **GIVEN** 玩家选择分支，**WHEN** 提交选择，**THEN** 正确传递给叙事事件系统
- **GIVEN** 事件处理完成，**WHEN** 关闭弹窗，**THEN** 主界面红点正确更新
- **GIVEN** 事件历史有20条，**WHEN** 打开历史列表，**THEN** 按时间倒序正确显示

### 性能测试

- **GIVEN** 打字机效果播放，**WHEN** 每帧更新，**THEN** 帧率保持60fps
- **GIVEN** 燃烧动画播放，**WHEN** 每帧更新，**THEN** 帧率保持60fps
- **GIVEN** 事件历史50条，**WHEN** 滚动列表，**THEN** 帧率保持60fps

## Open Questions

- **Q1**: 打字机效果是否应该支持"语速变化"——紧张场景打字更快，悲伤场景打字更慢？
- **Q2**: 事件卡片是否需要支持"收藏"——玩家可以标记喜欢的事件，方便回顾？
- **Q3**: 是否需要"快速跳过"功能——长按屏幕2秒跳过整个事件（直接显示选项）？
- **Q4**: 事件历史是否应该支持"搜索"功能？长期来看可能有价值，但MVP不需要。
- **Q5**: 纯叙事事件（无分支）是否也需要燃烧动画？当前设计是统一使用燃烧，但纯叙事事件的选择感较弱。
