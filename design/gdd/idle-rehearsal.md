# 放置排练系统 (Idle Rehearsal System)

> **Status**: In Design
> **Author**: [user + agents]
> **Last Updated**: 2026-04-23
> **Implements Pillar**: 成长是可见的, 选择是有意义的, 节奏是轻松的

## Overview

放置排练系统是《地下排练室》的**核心循环引擎**，负责将玩家的策略选择（曲目、队员、排练重点）转化为可量化的角色成长。它是连接"设置计划 → 放置等待 → 回来收获"这一核心闭环的心脏——没有它，乐队就只是一群站在地下室里无所事事的年轻人。

**系统边界**：
- **包含**：排练计划的结构与验证、排练效率计算、技能成长与熟练度成长计算、精力消耗规则、排练重点策略、化学反应触发判定、排练成果摘要生成
- **不包含**：角色状态的底层存储（属于角色属性系统）、曲目静态数据（属于曲目数据库）、离线时间计算（属于时间管理系统）、化学反应的具体事件内容（属于化学反应系统）、演出评分（属于演出判定系统）

**玩家如何感知**：
- 设置排练计划时权衡"技巧 or 默契 or 节省精力" → 策略感
- 离线回来看到"林野吉他+12，大伟鼓+8" → 成长感
- 看到"林野和大伟的默契度提升了" → 关系感
- 发现"精力充沛时成长更快" → 发现感

**为什么必须存在**：
没有放置排练系统，玩家就没有"做决策"的空间——游戏退化为纯点击收菜。它是"选择是有意义的"这一支柱的核心载体，也是玩家产生"我的安排产生了效果"这一控制感的来源。

## Player Fantasy

> **"你无法替他们弹奏，但你可以决定他们练什么、和谁练、练到多狠。然后你离开，信任时间。当你回来，地下室里的变化会告诉你：你的选择是对的，还是太温柔了，还是太残忍了。"**

你是这支乐队的隐形建筑师。不是指挥家——你不挥棒。不是经纪人——你不谈合同。你是那个在凌晨两点还在想"明天让林野继续磨那段 solo，还是让他和大伟合练培养默契"的人。然后你关掉手机去睡觉。

当你回来，你看到：
- 林野的吉他技能涨了，但他的心情跌了——他太倔了，不肯休息
- 大伟的鼓技能涨得慢，但他和林野之间多了一条新的事件线索
- 那首《锈》的熟练度从 35% 爬到了 47%——它开始像一首歌了

**核心情感时刻**：
你离线了 4 小时，回来时发现林野的精力耗尽了，但《锈》的熟练度刚好跨过 40% 门槛——解锁了新曲目《404》。你看着那个解锁提示，意识到：**这不是系统给你的奖励，这是你 4 小时前的决定在今天开花结果。**

**参考体验**：
- 《赛马娘》的训练安排——你不是在操作角色，你是在为角色创造条件
- 《足球经理》的战术设置——你的影响力是间接的、延迟的、但真实的

**这不是"最强乐队"的幻想——这是"我的影响力真实存在"的幻想。**

## Detailed Design

### Core Rules

**1. 排练计划结构**

玩家设置一个排练计划时，需要指定以下要素：

| 要素 | 必填 | 说明 |
|------|------|------|
| **曲目 (Song)** | 是 | 从已解锁曲目中选择一首 |
| **参与队员 (Members)** | 是 | 从可用角色中选择 1-4 人 |
| **排练重点 (Focus)** | 是 | 4 选 1：均衡 / 技巧 / 默契 / 节省 |

**计划有效性校验**：
- 若曲目所需乐器无对应角色覆盖：该乐器部分不产出技能成长（其他部分正常）
- 若某角色精力 ≤20：该角色无法参与排练，UI 提示"XX 累瘫了"
- 若所有角色精力均 ≤20：无法设置计划，提示"所有人都在睡觉"

**2. 排练执行模式**

| 模式 | 触发条件 | 计算方式 | 玩家可见 |
|------|----------|----------|----------|
| **实时排练** | 玩家在线，计划已确认 | 每 1 秒推进 1 分钟游戏时间 | 进度条实时推进 |
| **离线排练** | 玩家离开游戏 | 由时间管理系统调用本系统的离线计算接口 | 回来一次性显示结果 |

**两种模式使用完全相同的计算逻辑**，仅呈现方式不同。

**3. 精力消耗规则**

- 基础消耗：每人每小时 **10 点精力**
- 重点修正：
  - 均衡：1.0×（10/小时）
  - 技巧：1.2×（12/小时）
  - 默契：1.0×（10/小时）
  - 节省：0.7×（7/小时）

**精力不足处理**：
- 排练过程中，若某角色精力降至 ≤20：该角色立即停止排练，进入休息状态
- 其他角色继续排练（只要他们精力充足）
- 角色精力恢复至 >20 后，**不会自动恢复排练**（防止复杂的多轮循环逻辑）——本次排练中该角色已退出

**4. 技能成长计算**

每小时技能成长 = `skill_growth_per_hour[乐器] × rehearsal_efficiency × focus_multiplier`

- `skill_growth_per_hour`：来自曲目数据库
- `rehearsal_efficiency`：来自角色属性系统（0.0–1.56）
- `focus_multiplier`：
  - 均衡：1.0
  - 技巧：1.5
  - 默契：0.7
  - 节省：0.8

**每个角色独立计算**——同一排练中，林野和大伟的效率不同，成长也不同。

**5. 熟练度成长计算**

每小时熟练度成长 = `base_proficiency_per_hour × curve_multiplier × rehearsal_efficiency × focus_multiplier`

- `base_proficiency_per_hour`：来自曲目数据库
- `curve_multiplier`：来自曲目数据库（按曲线类型计算）
- `rehearsal_efficiency`：来自角色属性系统，使用**乐队平均效率**（所有参与角色的效率平均值）
- `focus_multiplier`：
  - 均衡：1.0
  - 技巧：0.75
  - 默契：1.0
  - 节省：0.8

**熟练度上限**：100%，硬封顶。

**6. 化学反应触发判定（MVP 简化版）**

每次排练结束时，检查一次化学反应触发：

```
触发概率 = base_chemistry_chance × compatibility_multiplier × focus_multiplier × mood_bonus
```

| 因子 | 说明 |
|------|------|
| `base_chemistry_chance` | 基础概率，MVP 固定 15% |
| `compatibility_multiplier` | 性格兼容性倍率（林野倔强+敏感 vs 大伟外向+随和 = 1.2×，具体由关系矩阵系统定义） |
| `focus_multiplier` | 默契重点=2.0×，其他=1.0× |
| `mood_bonus` | 两个角色都亢奋(+20%)=1.2×，都低落=0.5×，其他=1.0× |

**MVP 范围**：只实现林野+大伟的化学反应触发。其他角色组合预留接口。

**7. 排练成果摘要**

每次排练（无论实时还是离线）完成后，生成成果摘要：

| 字段 | 说明 |
|------|------|
| 总排练时长 | 实际排练小时数（非离线时长） |
| 曲目熟练度变化 | +X%（从 Y% 到 Z%） |
| 每角色技能成长 | 角色名: 乐器 +X |
| 精力变化 | 每角色: -Y |
| 心情变化 | 排练成功（有成长）= 心情 +5，排练无效（0 成长）= 心情 -5 |
| 化学反应触发 | 若有，显示"XX 和 YY 之间发生了些什么..." |
| 解锁检查 | 检查是否有新曲目/内容因本次成长而解锁 |

**8. 实时排练的特殊规则**

- 实时排练时，每 5 分钟游戏时间（即每 5 秒真实时间）执行一次"阶段性结算"：
  - 应用这 5 分钟的成长和消耗
  - 检查是否触发即时叙事事件（概率低，5%/阶段）
  - 更新 UI 进度条
- 玩家可随时点击"停止排练"中断，已完成的阶段收益保留
- 若玩家将游戏切入后台：实时排练自动暂停，下次回到前台时转换为离线计算

### States and Transitions

排练计划存在以下离散状态：

| 状态 | 触发条件 | 系统行为 |
|------|----------|----------|
| **未设置 (Idle)** | 无有效排练计划 | 乐队不进行任何活动，角色只进行自然状态变化（心情衰减、精力恢复） |
| **待执行 (Queued)** | 玩家已确认计划，但尚未开始 | 保存计划配置，等待时间推进 |
| **排练中 (Rehearsing)** | 时间开始推进（实时或离线） | 按小时/阶段计算成长、消耗、触发判定 |
| **完成待收 (Complete)** | 排练时长结束或玩家手动停止 | 生成成果摘要，等待玩家确认 |
| **已收获 (Harvested)** | 玩家点击确认成果 | 应用所有数值变化，触发后续系统（解锁检查、事件系统） |

```
        玩家设置计划                    时间推进
    ┌─────────────┐              ┌─────────────┐
    │             │─────────────►│             │
    │    未设置    │              │   排练中     │
    │             │◄─────────────│             │
    └─────────────┘   手动停止    └──────┬──────┘
           │                            │ 排练结束
           │                            ▼
           │                     ┌─────────────┐
           │                     │   完成待收   │
           │                     └──────┬──────┘
           │                            │ 玩家确认
           │                            ▼
           │                     ┌─────────────┐
           └────────────────────►│   已收获     │
                                 └─────────────┘
```

| 来源系统 | 输入数据 | 作用 |

| 关系矩阵系统 | 角色配对兼容性倍率 | 化学反应触发概率 |

**输出（本系统提供的数据）**：

| 目标系统 | 输出数据 | 作用 |
|----------|----------|------|

## Formulas

### 1. 每小时技能成长公式 (Hourly Skill Growth)

The hourly skill growth formula is defined as:

`skill_growth = skill_growth_per_hour × rehearsal_efficiency × focus_multiplier`

**Variables:**
| Variable | Symbol | Type | Range | Description |
|----------|--------|------|-------|-------------|
| skill_growth_per_hour | growth | float | 1–10 | 曲目定义的基础技能成长/小时 |
| rehearsal_efficiency | eff | float | 0.0–1.56 | 角色当前排练效率（来自角色属性系统） |
| focus_multiplier | focus | float | 0.7–1.5 | 排练重点倍率 |

**Focus Multiplier 表：**
| 重点 | focus_multiplier |
|------|------------------|
| 均衡 | 1.0 |
| 技巧 | 1.5 |
| 默契 | 0.7 |
| 节省 | 0.8 |

**Output Range:** 0 到 15.0（10 × 1.56 × 1.0，向下取整后 0–15）
**Example:** 曲目成长 4/hr，林野效率 1.3，技巧重点 → `skill_growth = 4 × 1.3 × 1.5 = 7.8`（向下取整 = 7）

### 2. 每小时熟练度成长公式 (Hourly Proficiency Growth)

The hourly proficiency growth formula is defined as:

`proficiency_growth = base_proficiency_per_hour × curve_multiplier × band_avg_efficiency × focus_multiplier`

**Variables:**
| Variable | Symbol | Type | Range | Description |
|----------|--------|------|-------|-------------|
| base_proficiency_per_hour | base | float | 1.0–10.0 | 曲目定义的基础熟练度成长率（%/小时） |
| curve_multiplier | curve | float | 0.5–1.3 | 熟练度曲线修正（来自曲目数据库） |
| band_avg_efficiency | avg_eff | float | 0.0–1.56 | 参与角色的平均排练效率 |
| focus_multiplier | focus | float | 0.75–1.0 | 排练重点倍率（技巧=0.75, 节省=0.8, 其他=1.0） |

**Output Range:** 0% 到 13.0%/小时（10 × 1.3 × 1.0 × 1.0）
**Example:** 《锈》base=4%/hr，曲线修正 0.9，乐队平均效率 1.1，均衡重点 → `proficiency_growth = 4 × 0.9 × 1.1 × 1.0 = 3.96%`

### 3. 化学反应触发概率公式 (Chemistry Trigger Chance)

The chemistry trigger probability formula is defined as:

`trigger_chance = base_chance × compatibility × focus_bonus × mood_bonus`

**Variables:**
| Variable | Symbol | Type | Range | Description |
|----------|--------|------|-------|-------------|
| base_chance | base | float | 0.15 | 基础触发概率（MVP 固定） |
| compatibility_multiplier | compat | float | 0.5–1.5 | 性格兼容性倍率（来自关系矩阵系统） |
| focus_bonus | focus | float | 1.0–2.0 | 默契重点=2.0，其他=1.0 |
| mood_bonus | mood | float | 0.5–1.2 | 都亢奋=1.2，都低落=0.5，其他=1.0 |

**Output Range:** 3.75% 到 54%（0.15 × 1.5 × 2.0 × 1.2）
**Example:** 林野+大伟 compatibility=1.2，均衡重点，林野亢奋、大伟正常 → `trigger_chance = 0.15 × 1.2 × 1.0 × 1.0 = 18%`

### 4. 精力消耗公式 (Energy Cost)

The energy cost formula is defined as:

`energy_cost = base_cost_per_hour × cost_multiplier × rehearsal_hours`

**Variables:**
| Variable | Symbol | Type | Range | Description |
|----------|--------|------|-------|-------------|
| base_cost_per_hour | base | int | 10 | 每小时基础消耗（来自角色属性系统） |
| cost_multiplier | cost_mult | float | 0.7–1.2 | 重点修正：节省=0.7, 均衡/默契=1.0, 技巧=1.2 |
| rehearsal_hours | hours | float | 0–6 | 实际排练小时数 |

**Output Range:** 0 到 72（10 × 1.2 × 6）
**Example:** 技巧重点，排练 3 小时 → `energy_cost = 10 × 1.2 × 3 = 36`

### 5. 实际排练时长公式 (Effective Rehearsal Hours)

The effective rehearsal hours formula is defined as:

`effective_hours = Σ(min(hour, energy_available / cost_per_hour))`

**Variables:**
| Variable | Symbol | Type | Range | Description |
|----------|--------|------|-------|-------------|
| hour | h | int | 1–N | 当前小时索引 |
| energy_available | energy | int | 0–100 | 角色当前精力 |
| cost_per_hour | cost | float | 7–12 | 每小时消耗（含重点修正） |

**说明**：逐小时检查——若角色精力足够，该小时计入排练；若不足，该小时为休息，不计入排练。

## Edge Cases

- **If 某角色在排练中途精力耗尽**：该角色立即停止参与，已完成的阶段收益保留。其他角色继续。
- **If 所有角色同时精力耗尽**：排练提前结束，总时长以实际排练小时计。
- **If 离线时长 < 1 分钟**：视为未排练，角色只进行自然状态变化。
- **If 离线期间没有设置排练计划**：成长收益为 0，角色只进行自然状态变化。
- **If 曲目熟练度达到 100% 后仍继续排练**：熟练度不再增长，但技能成长继续。UI 提示"已精通，继续排练只提升技能"。
- **If 化学反应触发时关系矩阵系统尚未实现**：使用预设的兼容性值（林野+大伟=1.2），记录待办事项。
- **If 排练计划中的曲目在排练期间被删除**：使用排练开始时的快照数据完成计算，不受后续修改影响。
- **If 玩家切换排练重点后立刻离线**：使用最后确认的重点配置。
- **If 角色处于"亢奋"状态但精力 < 20**：精力状态优先——角色无法参与排练。亢奋加成在精力恢复后生效。
- **If 玩家同时设置多个排练计划**：MVP 不支持队列——一次只能有一个计划。完成当前计划后才能设置下一个。
- **If 离线期间角色心情从正常跌至低落**：离线前锁定的效率值保持不变（简化处理）。效率重新计算发生在下次登录时。

## Dependencies

**上游依赖**：
| 系统 | 依赖数据 | 必要性 |
|------|----------|--------|
| 角色属性系统 | 技能值、状态值、性格标签、排练效率公式 | 硬依赖 — 成长和可用性计算的核心输入 |
| 曲目数据库 | 曲目参数、成长曲线、解锁条件 | 硬依赖 — 排练内容和收益的定义 |

| 关系矩阵系统 | 角色配对兼容性倍率 | 软依赖 — MVP 可用预设值替代 |

**下游被依赖**：
| 系统 | 依赖数据 | 数据流向 |
|------|----------|----------|
| 角色属性系统 | 更新后的技能值、状态值 | 放置排练 → 角色状态更新 |
| 曲目数据库 | 更新后的熟练度 | 放置排练 → 曲目熟练度更新 |

| 叙事事件系统 | 排练结果摘要 | 放置排练 → 事件触发条件 |

## Tuning Knobs

| 参数名 | 当前值 | 安全范围 | 过低效果 | 过高效果 | 关联系统 |
|--------|--------|----------|----------|----------|----------|
| `base_energy_cost_per_hour` | 10 | 5–20 | 角色几乎不消耗精力，可无限排练 | 角色快速耗尽，离线收益大幅减少 | 角色属性 |
| `base_chemistry_chance` | 0.15 | 0.05–0.30 | 化学反应太稀有，系统存在感弱 | 化学反应太频繁，失去特殊感 | 化学反应 |
| `focus_technical_skill_mult` | 1.5 | 1.2–2.0 | 技巧重点无吸引力 | 均衡和默契无人问津 | — |
| `focus_technical_prof_mult` | 0.75 | 0.5–0.9 | 技巧重点对熟练度惩罚太轻 | 技巧重点几乎不涨熟练度 | — |
| `focus_chemistry_skill_mult` | 0.7 | 0.5–0.9 | 默契重点成长惩罚太轻 | 默契重点几乎无技能成长 | 化学反应 |
| `focus_chemistry_trigger_mult` | 2.0 | 1.5–3.0 | 默契重点对化学反应加成不足 | 选择默契=必触发，失去随机性 | 化学反应 |
| `focus_energy_save_cost_mult` | 0.7 | 0.5–0.8 | 节省精力效果不明显 | 玩家永远选节省，其他重点无意义 | — |
| `focus_energy_save_growth_mult` | 0.8 | 0.6–0.9 | 节省重点成长惩罚太轻 | 节省重点成长过慢，无人使用 | — |

## Visual/Audio Requirements

- **排练进度条**：像一根被慢慢烧焦的琴弦——从银白到铁锈红的渐变表示进度。实时排练时，进度条每 5 分钟推进一小段，伴随轻微的走调拨弦声（越来越接近音准）。
- **排练完成效果**：进度条"烧断"时，一声终于对准调的吉他扫弦，像灯泡突然亮起。
- **技能成长数字**：像被墨水写在废纸上的数字，逐个浮现，有轻微的晕染效果。
- **熟练度进度**：曲目卡片上的熟练度条像被墨水慢慢填满的手撕纸条。
- **精力耗尽提示**：角色头像变灰，边缘出现裂纹效果，像一张被揉皱又展开的照片。
- **化学反应触发**：若排练中触发了化学反应，屏幕角落闪过一道霓虹色的光（垃圾摇滚美学中唯一的"荧光时刻"），伴随一声不和谐的鼓点。

## UI Requirements

- **排练计划设置面板**：
  - 曲目选择：横向滚动的曲目卡片，每张显示名称、难度标签、当前熟练度
  - 队员选择：角色头像拖拽到乐器位置，或自动分配
  - 重点选择：4 个涂鸦风格的按钮（均衡/技巧/默契/节省），选中时有墨迹扩散效果
  - 预览区域：显示预估收益（"预计吉他+5/小时，熟练度+3%/小时"）和预估精力消耗
  - 确认按钮：手写风格的"开练"按钮

- **实时排练界面**：
  - 中央：排练进度条（烧焦琴弦效果）
  - 下方：参与角色头像，显示实时精力变化
  - 右侧：当前曲目信息，熟练度实时更新
  - 停止按钮：随时可点击中断

- **排练成果弹窗**（离线回来或实时结束后）：
  - 标题："排练结束"（手写风格）
  - 总时长、曲目熟练度变化
  - 每角色的技能成长和精力变化
  - 化学反应触发提示（如有）
  - 解锁提示（如有）
  - "确认"按钮

## Acceptance Criteria

- **GIVEN** 林野吉他技能40、效率1.3、精力60，《锈》技能成长4/hr，**WHEN** 设置技巧重点排练1小时，**THEN** 吉他技能+7（4×1.3×1.5=7.8→7），精力-12
- **GIVEN** 乐队平均效率1.1，《锈》熟练度成长4%/hr、曲线修正0.9，**WHEN** 排练2小时，**THEN** 熟练度+7.92%（约+8%）
- **GIVEN** 林野精力15、大伟精力60，**WHEN** 设置两人一起排练，**THEN** 林野无法参与（精力≤20），大伟正常排练
- **GIVEN** 林野+大伟兼容性1.2、均衡重点、两人心情正常，**WHEN** 排练结束，**THEN** 化学反应触发概率=18%（0.15×1.2×1.0×1.0）
- **GIVEN** 默契重点排练1小时，**WHEN** 计算化学反应触发概率，**THEN** 概率翻倍（focus_bonus=2.0）
- **GIVEN** 曲目熟练度已达100%，**WHEN** 继续排练，**THEN** 熟练度不再增长，技能正常成长
- **GIVEN** 离线3小时、精力循环算法判定实际排练2.5小时，**WHEN** 计算成长，**THEN** 按2.5小时计算，不是3小时
- **GIVEN** 没有设置排练计划，**WHEN** 离线4小时，**THEN** 成长收益为0，角色只进行自然状态变化
- **GIVEN** 实时排练中每5分钟结算一次，**WHEN** 排练15分钟，**THEN** 执行3次阶段性结算
- **GIVEN** 玩家将游戏切入后台，**WHEN** 回到前台，**THEN** 实时排练自动转为离线计算，已完成的阶段保留

## Open Questions

- **Q1**: MVP 是否支持排练计划队列（排好下一首），还是每次只能有一个计划？
- **Q2**: 实时排练的"每5分钟阶段性结算"是否过于频繁？是否应该改为每15分钟或30分钟？
- **Q3**: 化学反应触发概率的安全范围（3.75%–54%）是否太宽？需要通过 playtest 验证实际感受。
- **Q4**: 精力耗尽后角色是否应该在本次排练中恢复后自动恢复参与？当前设计为"不恢复"以简化逻辑。
- **Q5**: 是否需要"自动排练"功能——当当前计划完成后自动重复相同的计划？
