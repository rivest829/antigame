# 排练计划UI (Rehearsal Planning UI)

> **Status**: Designed (pending review)
> **Author**: [user + agents]
> **Last Updated**: 2026-04-29
> **Implements Pillar**: 选择是有意义的

## Overview

排练计划UI是《地下排练室》的**核心交互界面**，是玩家最频繁使用的屏幕——每次登录后，玩家要么在设置新的排练计划，要么在查看当前排练状态。它是"选择是有意义的"这一支柱最直接的交互载体：玩家在这里做出的每一个选择（曲目、队员、排练重点）都会通过放置排练系统转化为具体的成长结果。

**系统边界**：
- **包含**：排练计划的设置界面布局、交互流程、预览显示、确认校验
- **不包含**：排练实时进行界面（属于主界面仪表盘）、排练成果展示（属于主界面弹窗）、角色详情展示（属于角色详情UI）、化学反应的详细计算（属于化学反应系统）

**玩家如何感知**：
- 横向滚动的曲目卡片，像一叠被翻烂的歌谱 → "这些是我们的歌"
- 拖拽角色头像到乐器位置 → "我在决定谁弹什么"
- 四个涂鸦风格的排练重点按钮 → "策略选择"
- 实时更新的预览数字 → "我的选择产生了可预测的后果"

**为什么必须存在**：
没有排练计划UI，玩家无法与核心循环交互。放置排练系统知道"怎么计算成长"，但它不知道"玩家想练什么"。排练计划UI是玩家意图的翻译器——把玩家的拖拽、点击、选择翻译成放置排练系统能理解的"计划数据结构"。

## Player Fantasy

> **你打开手机，屏幕上是地下排练室的俯视图。墙角的音箱积着灰，地板上散落着拨片和鼓棒。**
>
> 屏幕下半部分是一排曲目卡片。最左边是《地下室之光》——熟练度87%，卡片边缘已经起毛了，像被翻过很多次的旧谱子。右边是《锈》——熟练度55%，卡片上有一小块咖啡渍。再右边是一张灰色的卡片，上面写着"???"——你还没解锁这首。
>
> 你点了《锈》。卡片放大，占据了屏幕中央。
>
> 下方出现了两个空位：吉他位和鼓位。你点了一下林野的头像，拖到吉他位。林野的卡通形象抱着吉他，表情有点不耐烦——他的精力还剩45%，头像边缘有淡淡的黄色。你又点了大伟，拖到鼓位。大伟的精力还剩62%，头像在笑。
>
> 再下方是四个按钮：均衡、技巧、默契、节省。每个按钮都是手写的马克笔风格。你点了"默契"——按钮周围出现了一圈墨迹扩散效果。旁边的预览区更新："预计触发化学反应概率：35%"，"预计熟练度+2.5%/小时"。
>
> 你盯着那个35%看了两秒。上次的排练选了"技巧"，没有触发化学反应。你犹豫了一下，没改。
>
> 最下方是一个大大的"开练"按钮——像用黑色马克笔写在胶带上的字，边缘有点晕开。你点了下去。
>
> 屏幕震动了一下，像有人把音箱的音量突然调大。然后你回到了主界面，看到排练状态区显示：《锈》排练中，预计4小时后完成。
>
> 你关掉手机，去上班了。
>
> **核心情感时刻**：当你把大伟拖到鼓位时，他的头像旁边浮现了一行小字："大伟今天心情不错，鼓点可能会更有感觉。"这不是系统提示——这是大伟的"角色语音"。你在设置排练计划时，不仅在调整数字，你还在感受这些人是活着的。

**这不是"调设置"的幻想——这是"做安排"的幻想。** 你不是在配装，你是在为一群真实的人安排他们的下午。

## Detailed Design

### Core Rules

#### 1. 界面布局（三区结构）

排练计划UI采用垂直三区布局，从上到下依次为：

**A区：曲目选择区**
- 位置：屏幕上半部分（约占40%高度）
- 内容：横向滚动的曲目卡片列表
- 每张卡片显示：曲目封面、曲目名称、难度标签、当前熟练度百分比
- 已解锁曲目：彩色，可点击
- 未解锁曲目：灰色，显示锁定图标和解锁条件提示
- 选中曲目：卡片放大，显示详细信息（所需乐器、技能要求、预计收益）

**B区：队员分配区**
- 位置：屏幕中部（约占30%高度）
- 内容：
  - 上方：参与队员选择栏（可拖拽的角色头像池）
  - 下方：乐器位置槽（根据曲目所需乐器动态生成）
- 交互：拖拽头像到乐器槽，或点击头像后点击乐器槽
- 自动分配：点击"自动分配"按钮，系统按"技能最高→对应乐器"规则自动填充
- 角色状态：头像下方显示精力值，≤20时头像灰显，不可拖拽

**C区：重点选择与预览区**
- 位置：屏幕下半部分（约占30%高度）
- 内容：
  - 四个排练重点按钮：均衡 / 技巧 / 默契 / 节省
  - 预览面板：预计技能成长/小时、预计熟练度成长/小时、预计精力消耗、化学反应触发概率（如有）
  - 确认按钮："开练"

#### 2. 曲目卡片设计

```
┌──────────────────────┐
│ [曲目封面]            │
│                      │
│ 锈                   │
│ 难度: 普通           │
│ 熟练度: 55%          │
│ [▓▓▓▓▓░░░░░]        │
└──────────────────────┘
```

**卡片状态**：
- 默认状态：正常大小，可点击
- 选中状态：放大1.2倍，边框加粗，显示详细信息
- 未解锁状态：灰色，半透明，显示锁定图标
- 已精通状态（熟练度100%）：卡片边缘有金色描边

**卡片交互**：
- 点击：选中该曲目
- 左右滑动：滚动列表
- 长按：显示曲目详情弹窗（技能要求、历史最高评分、排练次数）

#### 3. 队员分配交互

**乐器槽设计**：
- 根据曲目所需乐器动态生成槽位
- 每个槽位显示：乐器图标 + 乐器名称 + 技能要求值
- 槽位状态：
  - 空槽：虚线边框，显示"拖拽角色到此处"
  - 已填充：显示角色头像 + 角色名称 + 当前技能值
  - 技能不足：头像边框变红，显示"技能不足"提示

**拖拽交互**：
1. 手指按下角色头像（≥100ms）→ 头像放大并跟随手指
2. 拖拽到乐器槽上方 → 槽位高亮
3. 释放 → 角色分配成功，头像缩小落入槽位
4. 拖拽到无效区域释放 → 头像弹回原位

**点击分配（替代交互）**：
1. 点击角色头像 → 头像高亮，进入"选中"状态
2. 点击空乐器槽 → 角色分配成功
3. 再次点击已选中角色 → 取消选中

#### 4. 排练重点选择

四个按钮采用涂鸦风格：

| 重点 | 图标 | 选中效果 | 预览变化 |
|------|------|---------|---------|
| 均衡 | 天平 | 墨迹扩散 | 成长均衡，无偏向 |
| 技巧 | 吉他solo剪影 | 墨迹扩散 | 技能成长↑，熟练度成长↓ |
| 默契 | 两个重叠的人形 | 墨迹扩散 + 霓虹光效 | 化学反应概率↑，技能成长↓ |
| 节省 | 电池图标 | 墨迹扩散 | 精力消耗↓，成长略降 |

**选中规则**：
- 单选，同一时间只能选中一个重点
- 切换重点时，预览数值实时更新（带0.3秒防抖）

#### 5. 预览面板

预览面板显示以下信息：

```
预计收益（每小时）:
├── 技能成长
│   ├── 林野·吉他: +5.2
│   └── 大伟·鼓: +3.8
├── 熟练度: +2.5%
├── 精力消耗
│   ├── 林野: -12/小时
│   └── 大伟: -10/小时
└── 化学反应概率: 35%（默契重点加成）
```

**预览更新规则**：
- 曲目/队员/重点任一变化 → 触发预览重算
- 防抖延迟：300ms（避免频繁切换时的计算抖动）
- 数值动画：从旧值滚动到新值，持续200ms

#### 6. 确认校验

点击"开练"按钮前，系统执行以下校验：

| 校验项 | 失败表现 | 提示文案 |
|--------|---------|---------|
| 已选择曲目 | 未选曲目 | "先选一首要练的歌" |
| 已分配队员 | 无队员 | "至少得有人排练吧" |
| 队员精力充足 | 所有角色精力≤20 | "所有人都在睡觉" |
| 乐器覆盖 | 某乐器无人覆盖 | "没人弹[乐器]，这首曲子少了点东西" |

**警告（不阻止确认）**：
- 某角色技能低于曲目要求："[角色]的[技能]可能不够用，确定要继续吗？"
- 某角色精力≤30："[角色]很累，排练效率会降低"

#### 7. 确认后流程

```
点击"开练"
  → 校验通过？
    → 否：显示错误提示，停留当前界面
    → 是：
      → 生成排练计划数据结构
      → 传递给放置排练系统
      → 显示"排练已开始"动画
      → 返回主界面
```

### States and Transitions

排练计划UI作为界面系统，有以下界面状态：

| 状态 | 说明 | 转换条件 |
|------|------|---------|
| **曲目选择 (Song_Select)** | 显示曲目列表，等待玩家选择 | 默认进入状态 |
| **队员分配 (Member_Assign)** | 已选曲目，等待分配队员 | 选择曲目后 |
| **重点选择 (Focus_Select)** | 已分配队员，等待选择重点 | 分配队员后（可与上一步合并） |
| **预览确认 (Preview_Confirm)** | 所有选择完成，显示预览 | 自动进入 |
| **确认中 (Confirming)** | 点击"开练"，执行校验 | 点击确认按钮 |
| **完成 (Done)** | 校验通过，计划已提交 | 校验通过后 |

```
曲目选择 → 队员分配 → 重点选择 → 预览确认 → 确认中 → 完成
   ↑___________________________________________________|
   （点击"重新开始"可返回曲目选择）
```

## Formulas

### 1. 预览数值计算 (Preview Calculation)

预览数值基于当前选择实时计算，使用与放置排练系统相同的公式：

```
skill_growth_preview = skill_growth_per_hour × rehearsal_efficiency × focus_multiplier
proficiency_growth_preview = base_proficiency_per_hour × curve_multiplier × band_avg_efficiency × focus_multiplier
energy_cost_preview = base_cost_per_hour × cost_multiplier
chemistry_chance_preview = base_chance × compatibility × focus_bonus × mood_bonus
```

**Variables:**
| Variable | Symbol | Type | Range | Description |
|----------|--------|------|-------|-------------|
| skill_growth_per_hour | sg | float | 1-10 | 曲目定义的基础技能成长 |
| rehearsal_efficiency | eff | float | 0.0-1.56 | 角色排练效率 |
| focus_multiplier | fm | float | 0.7-1.5 | 排练重点倍率 |
| base_proficiency_per_hour | bp | float | 1.0-10.0 | 曲目基础熟练度成长 |
| curve_multiplier | cm | float | 0.5-1.3 | 熟练度曲线修正 |
| band_avg_efficiency | avg | float | 0.0-1.56 | 乐队平均效率 |
| base_cost_per_hour | cost | int | 10 | 每小时基础精力消耗 |
| cost_multiplier | cmult | float | 0.7-1.2 | 重点修正 |
| base_chance | bc | float | 0.15 | 化学反应基础概率 |
| compatibility | compat | float | 0.5-1.5 | 性格兼容性 |
| focus_bonus | fb | float | 1.0-2.0 | 默契重点加成 |
| mood_bonus | mb | float | 0.5-1.2 | 心情加成 |

**注意**：预览数值仅供参考，实际成长受实时状态影响（如排练中途精力耗尽）。

### 2. 自动分配算法 (Auto-Assign Algorithm)

```
FOR EACH instrument_slot IN song.required_instruments:
    candidates = filter(characters, can_play(instrument))
    IF candidates is empty:
        leave_empty(slot)
    ELSE:
        best = max(candidates, by: skill(instrument))
        assign(best, slot)
```

**优先级**：技能值高 > 精力充足 > 心情好

## Edge Cases

- **If 无已解锁曲目**：显示"还没有可排练的曲目"，引导玩家查看解锁条件或返回主界面。
- **If 所有角色精力≤20**："开练"按钮禁用，显示"所有人都在睡觉"。提供"等待角色恢复"选项（跳转到主界面）。
- **If 横屏/竖屏切换**：竖屏时三区纵向排列，横屏时A区在左（40%宽度），B区和C区在右上下排列。
- **If 玩家快速连续切换曲目**：预览计算防抖（300ms），避免频繁计算导致卡顿。
- **If 拖拽过程中角色状态变化**（如网络同步导致精力更新）：使用拖拽开始时的快照状态，不中断当前交互。
- **If 选中曲目后解锁了新曲目**：曲目列表自动刷新，新曲目以动画形式插入列表。
- **If 玩家同时选择多个排练重点**：单选限制，切换时取消前一个选中状态。
- **If 化学反应系统未实现**：化学反应概率预览显示"???"，tooltip提示"需要更多排练数据"。
- **If 角色头像拖拽超出屏幕边界**：头像跟随手指到边界即止，释放后判断是否在有效槽位上方。

## Visual/Audio Requirements

### 视觉需求

**整体风格**：像一张被咖啡渍弄脏、被胶带修补过的排练表。背景是泛黄的纸张纹理，UI元素像是用马克笔和胶带贴上去的。

**曲目卡片**：
- 默认：正常大小，轻微阴影
- 选中：放大1.2倍，边框加粗，阴影加深
- 未解锁：灰色，半透明，锁定图标
- 过渡动画：缩放200ms + 阴影渐变200ms

**拖拽交互**：
- 按下：头像放大到1.3倍，透明度降至0.8
- 拖拽：头像跟随手指，下方有阴影
- 到有效槽位上方：槽位高亮（绿色描边）
- 到无效区域：槽位无反应
- 释放到有效槽位：头像缩小落入槽位，"咔哒"动画
- 释放到无效区域：头像弹回原位（弹簧动画）

**重点按钮**：
- 默认：手写体文字，无背景
- 选中：文字周围有墨迹扩散效果（从中心向外晕染）
- 切换：旧按钮墨迹消退，新按钮墨迹扩散

**"开练"按钮**：
- 默认：黑色马克笔风格文字，胶带背景
- 按下：文字稍微偏移（像被按下去的印章）
- 禁用：灰色，文字模糊

### 音频需求

**曲目选择**：轻快的拨弦声（每次切换不同音高）
**拖拽开始**：轻微的"啵"声（像胶带被撕开）
**拖拽落入槽位**："咔哒"声（像夹子扣上）
**重点切换**：轻微的鼓刷扫弦声
**预览数值更新**：数字滚动时的"咔嗒"声（极轻）
**"开练"按下**：一声有力的"咚"（底鼓），像排练开始的信号

## Acceptance Criteria

### 功能测试

- **GIVEN** 已解锁2首曲目，**WHEN** 打开排练计划UI，**THEN** 正确显示2张彩色卡片 + 灰色锁定卡片
- **GIVEN** 选中《锈》，**WHEN** 拖拽林野到吉他位，**THEN** 林野正确分配到吉他位
- **GIVEN** 已分配队员，**WHEN** 点击"默契"重点，**THEN** 预览区的化学反应概率正确更新（显示加成）
- **GIVEN** 所有角色精力≤20，**WHEN** 点击"开练"，**THEN** 按钮禁用，提示"所有人都在睡觉"
- **GIVEN** 某乐器无人覆盖，**WHEN** 点击"开练"，**THEN** 显示警告但不阻止确认
- **GIVEN** 完成所有选择，**WHEN** 点击"开练"，**THEN** 正确生成计划并传递给放置排练系统

### 边界测试

- **GIVEN** 无已解锁曲目，**WHEN** 打开UI，**THEN** 显示空状态提示
- **GIVEN** 横屏模式，**WHEN** 打开UI，**THEN** 布局正确适配（左-右分栏）
- **GIVEN** 快速切换曲目（<300ms），**WHEN** 停止切换，**THEN** 预览只更新最后一次选择
- **GIVEN** 拖拽角色到屏幕外，**WHEN** 释放，**THEN** 角色弹回原位，不分配

### 集成测试

- **GIVEN** 从曲目数据库读取曲目列表，**WHEN** 展示UI，**THEN** 曲目信息正确显示
- **GIVEN** 从角色属性系统读取状态，**WHEN** 展示头像，**THEN** 精力值正确显示
- **GIVEN** 点击"开练"，**WHEN** 校验通过，**THEN** 计划正确传递给放置排练系统
- **GIVEN** 解锁新曲目，**WHEN** 重新打开UI，**THEN** 新曲目出现在列表中

### 性能测试

- **GIVEN** 10首曲目，**WHEN** 滚动列表，**THEN** 帧率保持60fps
- **GIVEN** 切换重点，**WHEN** 预览更新，**THEN** 延迟<300ms
- **GIVEN** 拖拽角色，**WHEN** 移动手指，**THEN** 头像跟随无延迟（<16ms）

## Open Questions

- **Q1**: 是否需要支持"排练计划模板"——玩家保存常用配置（曲目+队员+重点），一键加载？这对高频用户很有用。
- **Q2**: 队员分配是否支持"一个角色同时演奏多个乐器"？MVP只有2个角色，但未来3-4人时是否允许多乐器角色？
- **Q3**: 是否需要"推荐配置"功能——系统根据当前状态推荐最优的曲目/队员/重点组合？
- **Q4**: 化学反应概率预览是否应该显示具体数值（35%）还是模糊描述（"较高"）？数值更透明，但模糊更有神秘感。
- **Q5**: 是否需要"排练计划队列"——排好下一首，当前完成后自动开始？MVP不支持，但未来可能需要。
