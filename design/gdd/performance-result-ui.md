# 演出结果UI (Performance Result UI)

> **Status**: Designed (pending review)
> **Author**: [user + agents]
> **Last Updated**: 2026-04-29
> **Implements Pillar**: 成长是可见的

## Overview

演出结果UI是《地下排练室》的**核心闭环展示界面**——玩家前期所有决策（排练计划、角色搭配、场地选择）的最终结果在这里呈现。它是"成长是可见的"这一支柱的最强音：数字滚动、评级揭晓、收益展示，每一个环节都在告诉玩家"你的准备配得上这个结果"。

**系统边界**：
- **包含**：演出回想动画、评分维度展示、评级揭晓、收益展示、反馈文本、详情面板
- **不包含**：演出判定的具体计算（属于演出判定系统）、声誉收益计算（属于声誉系统）、解锁判定（属于进度解锁系统）

**玩家如何感知**：
- 5秒模糊画面 + 曲目音频片段 → "我好像真的在现场"
- 四个维度的分数条从0滚动到实际值 → "每个维度都有结果"
- 评级大字弹出 → "这是总评价"
- 收益数字浮现 → "我获得了什么"
- 反馈文本 → "我知道哪里好、哪里不好"

**为什么必须存在**：
没有演出结果UI，演出判定只是一次后台计算——数字变了、声誉涨了，但玩家完全无感。演出结果UI是把"后台结算"翻译成"前端仪式"的关键层。它是放置游戏"设置→等待→收获"循环中的"收获"时刻，是整个核心循环的情感高潮。

## Player Fantasy

> **你离线了6小时。屏幕上没有直接弹出结果——而是先暗了下来。**
>
> 然后，一段模糊的画面开始播放。你看到了校园开放舞台的后台——灯光昏暗，墙壁上贴着褪色的海报。你听到了《锈》的失真吉他声，但声音很远，像隔着一扇门。
>
> 然后你听到了观众的反应。先是稀稀落落的掌声，然后——安静。然后是零星的"还行吧"。
>
> 画面慢慢清晰。你看到了评分界面。
>
> 四个维度的进度条开始从0滚动：
> - 技巧：0 → 72（蓝色条，滚动时有"咔嗒"声）
> - 配合：0 → 65（绿色条）
> - 台风：0 → 58（紫色条）
> - 观众反应：0 → 62（橙色条）
>
> 每个条滚动0.5秒。4个条总共2秒。
>
> 滚动结束后，屏幕中央出现了评级大字：
> **"成功"**
>
> 白色，淡入，停留1秒。
>
> 然后下方浮现了收益：
> - 声誉 +12
> - 林野·吉他 +2
> - 大伟·鼓 +1
> - 《锈》熟练度 +3%
>
> 再下面是反馈：
> - 亮点："林野的吉他solo有进步"
> - 改进："配合还可以更默契，建议多排练"
> - 观众："还行吧，可以再来"
>
> 你盯着"配合 65分"看了一会儿。你知道原因——熟练度才55%，而且没有触发"倔强节奏"。
>
> 你点了"继续"。屏幕淡出，回到主界面。声誉条涨了一小截。
>
> **核心情感时刻**：当评级是"完美"时，整个屏幕会震动一下，像有人把音箱音量突然调到了最大。评级文字不是淡入的——它是砸进来的，像有人把海报拍在了屏幕上。然后你听到了观众欢呼声，虽然很短，但足够让你感到：**我做到了。**

**这不是"看分数"的幻想——这是"站在台上听观众反应"的幻想。** 你不是在查看系统结算，你是在拆开一封演出后的信。

## Detailed Design

### Core Rules

#### 1. 界面布局（六段式流程）

演出结果UI采用纵向六段式布局，从上到下依次为：

**A段：回想区域**
- 位置：屏幕顶部
- 内容：5秒可跳过动画
- 画面：模糊场地背景 + 乐器剪影
- 音频：曲目片段（失真、现场感）
- 跳过：点击屏幕任意位置跳过

**B段：评级区域**
- 位置：回想区域下方（或屏幕中央，如果跳过回想）
- 内容：评级大字 + 总分
- 动画：评级大字弹出（不同评级不同动画）

**C段：评分分解**
- 位置：评级区域下方
- 内容：4个维度的进度条
- 维度：技巧（蓝）、配合（绿）、台风（紫）、观众反应（橙）

**D段：收益区域**
- 位置：评分分解下方
- 内容：声誉、技能成长、熟练度成长
- 动画：数字从0滚动到实际值

**E段：反馈区域**
- 位置：收益区域下方
- 内容：亮点、改进建议、观众引语

**F段：按钮区**
- 位置：屏幕底部
- 按钮：继续 / 查看详情 / 分享

#### 2. 回想动画

**内容**：
- 背景：演出场地的模糊画面（根据场地变化）
- 前景：乐器剪影（根据参与角色变化）
- 音频：曲目的5秒片段（失真处理，模拟现场音响效果）

**评级影响**：
| 评级 | 画面清晰度 | 音频质量 | 色调 |
|------|-----------|---------|------|
| 完美 | 清晰，温暖 | 清晰，饱满 | 暖色调 |
| 出色 | 较清晰 | 较好 | 中性偏暖 |
| 成功 | 一般 | 一般 | 中性 |
| 平庸 | 较模糊 | 有杂音 | 偏冷 |
| 灾难 | 模糊，有噪点 | 嘈杂，失真严重 | 冷色调 |

**跳过机制**：
- 点击屏幕任意位置：立即跳过回想，进入评级区域
- 跳过后，评级区域从屏幕中央开始展示

#### 3. 评分条设计

**进度条样式**：
```
技巧  [▓▓▓▓▓▓▓▓░░░░░░░░░░░░░░░░░░░░] 72/100
配合  [▓▓▓▓▓▓▓▓▓▓░░░░░░░░░░░░░░░░░░░] 65/100
台风  [▓▓▓▓▓▓▓░░░░░░░░░░░░░░░░░░░░░░] 58/100
观众  [▓▓▓▓▓▓▓▓▓░░░░░░░░░░░░░░░░░░░░] 62/100
```

**颜色分级**：
- 0-39：红色 (#B7410E)
- 40-59：橙色 (#D4A843)
- 60-74：黄色 (#E8D44D)
- 75-89：浅绿 (#7CB342)
- 90-100：深绿 (#2E7D32)

**滚动动画**：
- 从0滚动到实际值
- 速度：0.5秒/维度
- 缓动：ease-out-cubic
- 音效：滚动时的"咔嗒"声（老式计数器）

#### 4. 评级揭晓

**评级文字**：

| 评级 | 文字 | 动画 | 视觉特效 |
|------|------|------|---------|
| 完美 | "完美" | 从屏幕中央砸入（缩放2→1） | 金色 + 震动 + 粒子爆发 |
| 出色 | "出色" | 从上方落下（带弹跳） | 银色 + 缩放效果 |
| 成功 | "成功" | 淡入 | 白色 + 轻微光晕 |
| 平庸 | "平庸" | 静态显示 | 灰色 + 无特效 |
| 灾难 | "灾难" | 从下方滑入（带抖动） | 暗红 + 屏幕碎裂效果 |

**总分显示**：
- 评级文字下方显示总分
- 格式："总分：XX"（如"总分：65"）
- 动画：评级文字出现后0.3秒，总分淡入

#### 5. 收益展示

**收益项**：

| 收益类型 | 显示格式 | 颜色 |
|----------|---------|------|
| 声誉 | 声誉 +X | 金色 |
| 技能成长 | 角色·乐器 +X | 蓝色 |
| 熟练度成长 | 曲目名 +X% | 绿色 |

**动画**：
- 收益项逐个浮现
- 每项从0滚动到实际值
- 间隔：0.3秒/项
- 总时长：约1-2秒

**升级提示**：
- 如果本次演出导致声誉升级：在收益区域上方显示等级提升动画
- 动画：报纸/海报飘落，新等级名称涂鸦风格显示
- 持续时间：2秒
- 同时显示新解锁内容（如有）

#### 6. 反馈文本

**反馈区域布局**：
```
┌─────────────────────────────────────────┐
│ 反馈                                     │
│                                         │
│ ✦ 亮点                                  │
│ "林野的吉他solo有进步"                  │
│                                         │
│ ✧ 改进                                  │
│ "配合还可以更默契，建议多排练"          │
│                                         │
│ "还行吧，可以再来"                      │
│         —— 观众                         │
└─────────────────────────────────────────┘
```

**文本样式**：
- 亮点：引号包裹，绿色调
- 改进：引号包裹，黄色调
- 观众引语：引号包裹，斜体，灰色，较小字号

#### 7. 详情面板

**触发**：点击"查看详情"按钮

**面板内容**：
```
┌─────────────────────────────────────────┐
│ 演出详情                    [关闭]      │
│                                         │
│ 场地：校园开放舞台                       │
│ 曲目：锈                                │
│ 成员：林野(吉他), 大伟(鼓)              │
│                                         │
│ 技巧计算：                              │
│ 技能覆盖率 116% × 熟练度加成 127%       │
│ × 乐器匹配 100% = 100（封顶）           │
│                                         │
│ 配合计算：                              │
│ 关系基础 65% × 化学反应加成 115%        │
│ × 熟练度加成 116% = 87.1                │
│                                         │
│ 难度修正：                              │
│ 原始总分 63.5 × 难度系数 0.90 = 57.15   │
└─────────────────────────────────────────┘
```

**展开动画**：从底部滑入，300ms，ease-out

#### 8. 演出取消状态

如果演出因角色状态极差而取消：

**界面变化**：
- 回想区域：显示"演出延期"，无音频
- 评级区域：显示"演出取消"
- 评分分解：隐藏
- 收益区域：显示"无收益"
- 反馈区域：显示取消原因（如"乐队状态太差，无法演出"）
- 按钮："返回排练" / "查看角色状态"

### States and Transitions

演出结果UI有以下展示状态：

| 状态 | 说明 | 转换条件 |
|------|------|---------|
| **回想 (Recall)** | 播放回想动画 | 进入结果界面时 |
| **评分滚动 (Score_Scroll)** | 4个维度分数滚动 | 回想结束或跳过后 |
| **评级揭晓 (Rating_Reveal)** | 评级大字弹出 | 评分滚动完成后 |
| **收益展示 (Reward_Display)** | 收益数字滚动 | 评级揭晓后 |
| **反馈展示 (Feedback_Display)** | 反馈文本显示 | 收益展示后 |
| **详情展开 (Details_Expanded)** | 详情面板打开 | 点击"查看详情" |
| **完成 (Done)** | 所有展示完成 | 自然结束 |

```
进入 → 回想 → 评分滚动 → 评级揭晓 → 收益展示 → 反馈展示 → 完成
        ↓（跳过）
      评分滚动 ───────────────────────────────→
```

**输入（本系统需要的数据）**：

| 来源系统 | 输入数据 | 作用 |

## Formulas

### 1. 回想动画时长 (Recall Animation Duration)

```
recall_duration = base_duration × skip_multiplier
```

**Variables:**
| Variable | Symbol | Type | Range | Description |
|----------|--------|------|-------|-------------|
| base_duration | base | int | 3000-8000ms | 基础回想时长 |
| skip_multiplier | skip | float | 0.0 | 跳过时为0 |

**Default**: 5000ms（可跳过）

---

### 2. 评分条滚动进度 (Score Bar Scroll Progress)

```
score_display = lerp(0, actual_score, ease_out_cubic(t))
where t = elapsed_time / scroll_duration
```

**Variables:**
| Variable | Symbol | Type | Range | Description |
|----------|--------|------|-------|-------------|
| scroll_duration | dur | int | 400-800ms | 单条滚动时长 |
| stagger_delay | delay | int | 200-400ms | 条与条之间的间隔 |

**Example**: 4条，每条500ms，间隔300ms
- 技巧条：0-500ms滚动
- 配合条：300-800ms滚动
- 台风条：600-1100ms滚动
- 观众条：900-1400ms滚动
- 总时长：约1.4秒

---

### 3. 评级动画强度 (Rating Animation Intensity)

```
animation_intensity = rating_intensity_map[rating]
```

| 评级 | 强度值 | 震动幅度 | 粒子数量 | 音效音量 |
|------|--------|---------|---------|---------|
| 完美 | 1.0 | 大 | 多 | 大 |
| 出色 | 0.7 | 中 | 中 | 中 |
| 成功 | 0.4 | 小 | 少 | 小 |
| 平庸 | 0.1 | 无 | 无 | 无 |
| 灾难 | 0.8 | 中（抖动） | 碎裂效果 | 中 |

## Edge Cases

- **If 玩家跳过回想后立刻跳过评分滚动**：评级区域直接显示最终分数（无滚动动画），评级文字直接显示。
- **If 演出评级为"灾难"且连续第3次**：结果UI顶部显示特殊提示"乐队陷入了危机……"，提供"查看事件"按钮（跳转到叙事事件UI）。
- **If 本次演出同时触发升级和新解锁**：先播放评级动画，再播放升级动画（顺序不可颠倒，避免信息过载）。
- **If 详情面板打开时玩家点击"继续"**：先关闭详情面板（滑出动画），然后退出结果UI。
- **If 演出结果数据不完整**（如某维度分数缺失）：该维度显示"N/A"，其他维度正常展示。
- **If 玩家连续快速点击"继续"**：忽略快速点击，只有当前动画/展示完成后才响应。
- **If 演出取消（非正常结果）**：跳过回想和评分滚动，直接显示"演出取消"页面。

|------|--------|---------|---------|

## Visual/Audio Requirements

### 视觉需求

**回想区域**：
- 背景：场地模糊画面（高斯模糊）
- 前景：乐器剪影（黑色，半透明）
- 滤镜：根据评级变化（完美=暖色调增强，灾难=冷色调+噪点）
- 过渡：从模糊到清晰的渐变（仅在回想结束时，如果未跳过）

**评分条**：
- 背景：深灰色轨道
- 填充：彩色渐变（根据分数区间变色）
- 数字：填充完成后，数字从0滚动到实际值
- 标签：维度名称在左侧，分数在右侧

**评级文字**：
- 完美：金色大字（#FFD700），字体粗犷，带轻微发光
- 出色：银色大字（#C0C0C0），字体略细
- 成功：白色大字（#FFFFFF），标准字体
- 平庸：灰色大字（#8C8C8C），字体模糊边缘
- 灾难：暗红大字（#8B0000），字体破碎效果

**粒子特效**（仅完美评级）：
- 从评级文字中心向四周爆发
- 粒子：金色星星 + 音符形状
- 数量：50-100个
- 持续时间：1秒

**屏幕震动**（完美评级）：
- 幅度：5px
- 频率：10Hz
- 持续时间：0.3秒

### 音频需求

**回想阶段**：
- 曲目片段（5秒）
- 完美/出色：清晰、饱满
- 成功：正常
- 平庸：有轻微杂音
- 灾难：嘈杂、失真严重

**评分滚动**：
- 每个维度滚动："咔嗒"声（老式计数器）
- 音调：随分数升高而升高

**评级揭晓**：
- 完美：完整和弦 + 观众欢呼（2秒）
- 出色：简短和弦 + 掌声（1秒）
- 成功：单音符 + 轻微掌声（0.5秒）
- 平庸：低沉的"嗯"（0.3秒）
- 灾难：不和谐音符 + 嘘声（1秒）

**收益展示**：
- 声誉增加：钹的轻击
- 技能增加：对应乐器的单音符

## Acceptance Criteria

### 功能测试

- **GIVEN** 演出评级为"成功"，**WHEN** 进入结果UI，**THEN** 4个维度分数正确显示，总分=加权平均
- **GIVEN** 评分滚动动画，**WHEN** 播放中，**THEN** 每个条从0滚动到实际值，0.5秒/条
- **GIVEN** 评级揭晓，**WHEN** 评分滚动完成，**THEN** 评级大字正确弹出（白色淡入）
- **GIVEN** 收益展示，**WHEN** 评级揭晓后，**THEN** 收益数字正确滚动显示
- **GIVEN** 点击"查看详情"，**WHEN** 按钮被点击，**THEN** 详情面板从底部滑入，显示计算过程
- **GIVEN** 演出取消，**WHEN** 进入结果UI，**THEN** 显示"演出取消"页面，无评分无收益

### 边界测试

- **GIVEN** 回想播放中，**WHEN** 点击屏幕，**THEN** 回想立即跳过，进入评分滚动
- **GIVEN** 连续3次灾难评级，**WHEN** 进入结果UI，**THEN** 顶部显示危机提示
- **GIVEN** 同时触发升级和新解锁，**WHEN** 展示收益，**THEN** 升级动画先于解锁动画播放
- **GIVEN** 详情面板打开，**WHEN** 点击"继续"，**THEN** 面板先关闭，然后退出结果UI

### 集成测试

- **GIVEN** 从演出判定系统读取结果数据，**WHEN** 展示UI，**THEN** 所有数据正确显示
- **GIVEN** 从声誉系统读取升级信号，**WHEN** 展示收益，**THEN** 升级动画正确触发
- **GIVEN** 结果查看完成，**WHEN** 返回主界面，**THEN** 主界面状态正确更新
- **GIVEN** 演出结果需要存档，**WHEN** 退出结果UI，**THEN** 结果正确保存

### 性能测试

- **GIVEN** 回想动画播放，**WHEN** 每帧更新，**THEN** 帧率保持60fps
- **GIVEN** 评分条滚动，**WHEN** 4条同时滚动，**THEN** 帧率保持60fps
- **GIVEN** 完美评级粒子特效，**WHEN** 100个粒子同时渲染，**THEN** 帧率保持60fps
- **GIVEN** 详情面板展开，**WHEN** 滑入动画，**THEN** 耗时<300ms

## Open Questions

- **Q1**: 回想动画是否应该在MVP中实现？5秒动画对留存有帮助，但增加了美术和音频资源需求。
- **Q2**: 是否需要"演出回放"功能——允许玩家重新观看演出的"模拟过程"？这会增加开发量，但增强沉浸感。
- **Q3**: 评分条的颜色分级是否需要支持色盲模式？当前使用颜色+文字双重编码，但色盲友好性需要验证。
- **Q4**: 详情面板是否应该支持"分享到社交媒体"？当前设计有"分享"按钮，但MVP可能不需要。
- **Q5**: 演出取消后，是否应该允许玩家"补救"——如消耗资源立即恢复角色状态重新演出？
